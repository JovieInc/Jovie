## Top-10 layout-shift risks (Sub-Agent 2)

1. **`/app/admin/activity`** — Suspense fallback uses raw `<table>` w/ 8 rows but loaded view is virtualized `UnifiedTable`. Header heights, row heights, and column widths all differ → visible jump on hydration. Class A+G. **Highest priority.**
2. **`/app/admin/investors`** — raw `<table>` w/ no fixed col widths inside `Suspense`; the `<TableSkeleton>` fallback has no width parity. First-paint columns auto-size, then re-layout when data arrives. Class A+G.
3. **`/app/admin/campaigns`** — every section in `InviteCampaignManager` independently spins; cumulative shift as each section finishes. Class A+G.
4. **`/app/admin/outreach?queue=dm`** — centered spinner replaces a 1/2/3-col card grid; once data arrives the page resizes vertically by hundreds of pixels. Class G.
5. **`/app/dashboard/presence`** — `loading.tsx` shows 3 card skeletons but the live page can be in card OR table mode; mode mismatch → shift. Class G+H.
6. **`/app/admin/people?view=users`** — Status, Suppression, Welcome columns are 130px each but content is a 22px badge or em-dash → first row paints sparse; if the underlying badge variant changes (Failed/Active/Suspended) cell content width can jump. Class D+G.
7. **`/app/admin/people?view=releases`** — Issues column uses `flex flex-wrap` w/ no min-height; first paint with empty issues vs loaded with 4 pills shifts row height by 16-20px. Class A+E.
8. **`/app/admin/people?view=creators`** — opening AdminProfileSidebar collapses table to narrower viewport without reserving width via `tableMeta.rightPanelWidth`. Verify and reserve. Class A+H.
9. **`/app/admin/outreach?queue=email`** — uses `ContentTable` molecule (not `UnifiedTable`) + spinner placeholder; same pattern as DM queue. Class G.
10. **`/app/admin/_components/AdminScoreboardSection`** — funnel steps use `min-w-[100px]` inside `overflow-x-auto`; on slow data load the funnel paints empty-state then grows. Class A+G.

## Top-10 shared primitive opportunities (Sub-Agent 2)

1. **`UnifiedTableSkeleton`** — single skeleton component that takes the same `columns: ColumnDef[]` array and renders 8 shimmer rows that match. Solves Activity, Investors, Outreach, Email Queue, Campaigns at once.
2. **`CompactStatusCell`** — right-aligned, 22px-min-height badge cell w/ reserved column width = `2 * cellPadding + maxLabelWidth`. Used by Users, Feedback, Leads, Tour Dates, Releases.
3. **`PillGroupCell`** — wraps a list of pills with `flex-wrap` and a `maxStackHeight` so first paint reserves at least one row height. Used by Leads Signals, Releases Issues, Waitlist Goals, Suggestions.
4. **`IssueIconStack` / `IconCluster`** — icon-only pill cluster with tooltip; replaces label+icon pill rows where labels are redundant. Used by Releases Issues, Leads Signals, DSP Presence.
5. **`ScoreCell`** — right-aligned tabular numeric w/ optional bar visual; replaces ad-hoc `<span className='tabular-nums'>` in Leads, Investors, Audience LTV.
