## Evidence policy

- High confidence requires direct imports or API use proving the boundary.
- Medium confidence is an evidence-backed inference that still needs owner review.
- Missing timing and flake data stays null with provenance; zero is never substituted for missing evidence.
- Live providers and external sandboxes stay off PR gates unless a risk trigger requires them.
- Generated inventory coverage is validated against every tracked executable test path.
