## Follow-up

Pickup agent must first judge whether to implement, close, or split this.

Build a data-driven loop to identify which model + prompt variant + content shape converts best per skill, per artist segment. Two viable substrates:

1. **Statsig** — Already integrated in the codebase (per recent commits). Gates per-skill model selection + prompt variant. Treat Save / Dismiss / Tweak as the conversion signal. Auto-recommend winners.
2. **Braintrust** — Purpose-built for LLM eval + experimentation. If it natively handles generative-quality eval (image/video, not just text), prefer it.

**PR2-time investigation step:** spike on Braintrust capabilities before deciding. If Braintrust does image/video gen eval natively, use it. If not, default to Statsig + custom logging.

## Why it matters

Skill quality is the kill-criterion. We need to know which model + prompt variant is actually winning, not guess. Data-driven model selection is also the moat Codex named in Phase 1 ("the defensible wedge is proprietary operating memory + measured revenue outcomes").
