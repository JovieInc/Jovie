# JOV-2533: Follow-up: use composite user+IP key for account email rate limit

- **Status:** Backlog
- **Priority:** 🔵 Low
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2533/follow-up-use-composite-userip-key-for-account-email-rate-limit
- **Updated:** 2026-05-31T04:57:41.788Z

The `/api/account/email` limiter added in the security sweep is currently IP-only. That is acceptable for the initial fix, but it can cause collateral throttling on shared NATs and is easier to bypass with IP rotation.

Follow-up idea:

* After auth resolves, prefer a composite key using the authenticated user ID plus client IP
* Keep the current IP-only fallback for anonymous or unauthenticated paths

This is a non-blocking improvement and should stay separate from the current small PR.
