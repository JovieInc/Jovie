[JovieInc/Jovie#9544](https://linear.app/jovie/issue/JOV-2547/public-profile-system-b-ios-polish-pass) covered the highest-traffic public profile surfaces and a few safe global token violations. Remaining read-only audit findings that should be handled in a separate scoped lane:

* Onboarding V1 shell uses raw dark colors, gradients, 28/32px radii, and custom shadows instead of System B tokens.
* Waitlist intake chat reimplements a separate dark UI system with raw colors and hand-rolled buttons.
* Auth fallback and desktop auth repeat raw dark shells, buttons, and marketing-scale headlines.
* Dashboard preview panel nests multiple framed card surfaces around the phone preview.
* Several non-profile app and experiment surfaces still use decorative hover/active scale or translate motion and should be converted to color/opacity states.

Ship now: keep [JovieInc/Jovie#9544](https://linear.app/jovie/issue/JOV-2547/public-profile-system-b-ios-polish-pass) focused on public-profile conversion surfaces and low-risk shared token fixes.  
Re-evaluate when: public-profile QA is green and this queued drift is the highest-impact visible System B inconsistency, or when the affected surfaces receive more than 5% of weekly app sessions.  
Then: run a dedicated app-wide System B sweep with visual QA for onboarding, waitlist, auth, dashboard preview, and remaining motion-token violations.
