## Source

* Parent: [JOV-2148](https://linear.app/jovie/issue/JOV-2148/hardened-coverage-for-human-eye-only-ui-inconsistencies)

## Follow-up

Build a Storybook story matrix that renders `Card`, `DrawerSurfaceCard`, `EntitySidebarShell`, and other elevation surfaces in every nesting combination defined in `.claude/rules/ui.md` "Surface Elevation Rules":

* Card on shell canvas
* Card inside Card (banned)
* Card inside DrawerSurfaceCard with variant flat
* Surface-1 on surface-1 without border (banned)
* etc.

Snapshot each combination with Playwright `toHaveScreenshot` in both light and dark modes.

## Why it matters

Gotcha class #8 — invisible cards (same color on same color, missing border, etc.) are the single most common visual regression we ship.

## Acceptance criteria

* Stories under `apps/web/.storybook/stories/elevation-matrix.stories.tsx`
* Playwright spec under `apps/web/tests/e2e/storybook-elevation.spec.ts` (or component-test runner)
* CI runs on every PR; failures block merge
* Banned combinations have explicit "this should look BROKEN" baseline as a sanity check

## Dependency

Blocks `Chromatic evaluation` ([JOV-2152](https://linear.app/jovie/issue/JOV-2152/candidate-follow-up-evaluate-chromatic-vs-expanded-playwright-visual)) decision — should run this matrix locally first to size effort.
