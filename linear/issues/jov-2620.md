## Follow-up

`pnpm --filter @jovie/web run qa:public:exhaustive` reaches the `root-not-found` public route probe and fails because the QA runner treats the expected 404 resource from `/dev/root-not-found-probe` as a critical console error.

## Why it matters

This blocks `/qa exhaustive` from completing even after the app builds and earlier public routes pass. It makes unrelated product work look broken and hides true regressions behind a QA harness false failure.

## Classification

Required

## Acceptance criteria or triage question

* Public route QA can test the root-not-found probe without treating its expected 404 resource as a critical console error.
* The exhaustive public QA command exits 0 when no real route regression is present.
