# JOV-1899: Build Electron Local HUD shell with hosted/local mode detection

- **Status:** Backlog
- **Priority:** 🟠 High
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** —

- **Updated:** 2026-05-31T02:17:52.486Z

Split the HUD into two runtimes that share the same UI shell:

## Electron Local HUD

* Launches like a normal Mac app
* Talks to local Hermes, GBrain, worktrees, crons, logs, scripts
* Can execute local commands safely (local-only)
* Detects local install and enables "control mode"

## Hosted Staging HUD

* Same UI shell
* Read-only / status-first
* Can create approval/dispatch manifests
* Cannot execute shell commands

## Architecture

```
shared HUD UI package
├─ web mode: /admin/ops on staging
└─ electron mode: local desktop app

local-only bridge
├─ Hermes control
├─ cron control
├─ worktree/agent status
├─ local logs
└─ GBrain/memory sync
```

Runtime detection:

```ts
const isElectron = Boolean(globalThis.electronAPI)
const isLocalHost = location.hostname === "localhost"
const controlMode = isElectron && isLocalHost
```

## Hard rule

* Hosted = observe + approve
* Electron/local = observe + execute

## Acceptance

* Same HUD UI runs in web and Electron
* Electron app opens local HUD without manually starting the dev server
* Local bridge exposes only allowlisted commands
* Hosted staging never exposes command execution
* UI clearly shows "Hosted read-only" vs "Local control mode"

## Dependencies

* Builds on the desktop scaffold from #8144
* Requires Hermes / Hermes-CLI dispatcher (already shipped via #8092)
