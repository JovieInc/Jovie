# JOV-2669: UX: Standardize app-wide search entry and header behavior

- **Status:** Todo
- **Priority:** 🟠 High
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** grok
- **URL:** https://linear.app/jovie/issue/JOV-2669/ux-standardize-app-wide-search-entry-and-header-behavior
- **Updated:** 2026-05-31T07:48:21.000Z

## Problem

Search entry points are inconsistent — some views have a search icon, some have a text input, header behavior varies.

## Scope

* Audit all search entry points across the app
* Standardize on one search trigger pattern (Cmd+K primary, header icon secondary)
* Unified search modal component
* Consistent header layout across views

## Acceptance

* Every view has the same search trigger
* Cmd+K works globally
* Header search icon opens same modal
