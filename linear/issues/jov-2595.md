# JOV-2595: Roadmap: consolidate fragmented UI flows into chat tools and connectors

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2595/roadmap-consolidate-fragmented-ui-flows-into-chat-tools-and-connectors
- **Updated:** 2026-05-30T23:17:42.155Z

Audit from the chat-first pitch conversion.

Candidate surfaces to consolidate into the agentic tools/skills/connectors system:

* Release plans: keep the task board as the system of record, but make plan generation a visible chat/action-menu skill with task-derived context instead of a wizard-first workflow.
* Canvas and lyrics workflows: promote format lyrics, Canvas plan, and Canvas uploaded state into release/task action-menu skills so chat can explain, execute, and record results.
* Album art: already chat/action-menu based; align catalog registration and output cards with the pitch pattern so generated assets, prompts, and approval states share one artifact contract.
* Profile/settings actions: retire duplicated settings-only AI affordances where equivalent chat tools exist, especially bio import, profile edits, avatar, and social link changes.
* Connector-draft actions: add Gmail draft support only when Gmail has write/draft scopes; keep default outputs copy-paste-safe until connector capability is present.
* Task actions: use task title/category/metadata as the first-class routing layer for skills, not one-off per-surface buttons.

EVENT: This is a product architecture cleanup, not a timing optimization. The interface should prefer one central agentic action system over repeated mini-builders.
