# JOV-2644: Standardize app-wide search entry and header behavior

- **Status:** Todo
- **Priority:** 🟠 High
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2644/standardize-app-wide-search-entry-and-header-behavior
- **Updated:** 2026-05-31T04:56:31.128Z

Standardize search UX across core screens so search always starts from the left rail and expands into a header input.

The intended pattern is:

* Search button in the left rail
* Activation opens a focused search input in the header
* Search filters the main content live
* The same interaction model works across Library, Audience, and tasks

This should replace the current inconsistent search placement and screen-specific search patterns.
