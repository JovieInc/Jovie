# JOV-2632: Redesign merch chat output into compact 3-card picker

- **Status:** In Progress
- **Priority:** 🔴 Urgent
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** grok
- **URL:** https://linear.app/jovie/issue/JOV-2632/redesign-merch-chat-output-into-compact-3-card-picker
- **Updated:** 2026-05-31T07:20:05.000Z

The merch generation result needs to behave like a compact merch picker rather than a verbose AI transcript.

Key problems:

* The result renders like one long generated block with embedded cards instead of a true 3-card grid.
* A large outer container visually competes with the cards.
* The UI uses double carding, including oversized confirmation states.
* User-facing cards expose internal pricing and provider/debug details.
* Warning copy is repeated per card instead of shown once globally.
* Card copy is too long and includes synthetic-sounding phrases.
* Titles are not optimized for scanning.
* Card hierarchy, CTA treatment, badge treatment, spacing, and image sizing are inconsistent.
* The chat repeats titles and exposes generation IDs.
* User-facing merch selection and admin/debug information are not clearly separated.

Requested behavior:

* Use a true 3-card bento layout:
  * Desktop: 3 equal cards in one row.
  * Tablet: 2 + 1 layout.
  * Mobile: single-column stacked cards.
* Render a concise section header followed by the 3 cards, without wrapping everything inside one oversized container.
* Each merch option should be a single visual surface.
* After selection, show a minimal confirmation row instead of another large card.
* Show one visible price only: `$45`.
* Hide artist, platform, floor, margin math, fulfillment cost, generation IDs, provider snapshot details, and production warnings from the normal user UI.
* Move internal economics, Printful status, warnings, and provider details into a separate admin/debug surface.
* Replace repeated provider warnings with one compact global notice: `Draft only. Connect Printful to publish.`
* Cap each card to:
  * Product name
  * One-line descriptor
  * Price
  * CTA
* Remove synthetic filler phrases from all user-facing copy.
* Rename cards for scannability:
  * `Signal Tee`
  * `Object Tee`
  * `Archive Tee`
  * Show the artist name once in the section header if needed.
* Use this hierarchy inside each card:
  * Image
  * Small `Draft` badge
  * Title
  * One-line description
  * Price
  * CTA row
* Use one primary CTA: `Save Draft`.
* Hide publish behind overflow or a disabled treatment with tooltip.
* Move `Option 1 / 2 / 3` to subtle eyebrow labels.
* Standardize image aspect ratio and dimensions.
* Compress vertical spacing so each card stays roughly 220–300px tall depending on image size.
* Use a concise header:
  * `Merch concepts`
  * `Pick one to save or ask for edits.`
* Do not repeat the chat title in the response.
* Selection confirmation should be minimal, e.g.:
  * `Saved: Signal Tee`
  * `Draft added to Library.`
  * Button: `Open Library`

Reference mock layout:

Merch concepts

Pick one to save or ask for edits.

\[Signal Tee\] \[Object Tee\] \[Archive Tee\]

Draft only. Connect Printful to publish.

Overall direction:

* The experience should feel compact, visual, and decision-oriented.
* Normal users should see design, name, price, and save action.
* Admin-only operational state should be collapsed, hidden, or moved to dedicated tooling.
