# JOV-2098: design(admin): consolidate overlapping IA across Overview / Ops / Growth / People screens

- **Project:** —

- **Updated:** 2026-05-31T02:16:26.204Z

## Problem

The admin nav has four top-level sections that overlap in content and purpose:

* **Overview** — high-level metrics, but some of these duplicate what's on Ops
* **Ops** — operational controls + Kanban, but also surfaces metrics that could live on Overview
* **Growth** — acquisition stats, but some growth metrics appear on Overview too
* **People** — user management, but some user stats appear on Growth

The result is that admins have to check multiple screens to get a complete picture of any single concern.

## Goal

Audit and map what each screen currently shows, identify duplication, and produce a clear IA proposal where each screen has a distinct, non-overlapping purpose:

* **Overview** — health dashboard (one number per key area, each linking to the detailed screen)
* **Ops** — live operational state + controls (Kanban, control panel, incidents)
* **Growth** — acquisition funnel, referral, conversion (no overlap with Overview)
* **People** — user table, role management, individual user actions

## Acceptance criteria

* IA audit document or Linear comment mapping current → proposed
* No metric or control appears on more than one screen
* Navigation labels and screen titles match the new IA
* No screens are removed without replacement routing
