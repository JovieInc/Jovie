v1 magic moment for touring DJs (40+ shows/year): Gmail booking email → suggested Google Calendar event → DJ approves → calendar entry created.

Concierge-first per /autoplan review with four reviewers (Codex + Claude CEO + Claude Eng + Claude DX).

### Persona

Touring DJ playing 40+ shows/year. Calendar pain is real; Jovie's public tour page is the canonical fan-facing surface.

### Key architectural decisions (from review)

* New `workflow_runs` table — `ingestionJobStatusEnum` cannot host `waiting_for_approval` (verified at `apps/web/lib/db/schema/enums.ts:132`).
* CAS state transitions (`pending→approved`, `approved→executed`) + deterministic Google `event.id` = `suggested_actions.id` for at-most-once execution.
* AI Gateway (`@ai-sdk/gateway`) not raw `@anthropic-ai/sdk`. Model `claude-sonnet-4-6`.
* CSP allowlist in `apps/web/lib/security/content-security-policy.ts` `connectSrcBase`, NOT `cdn-domains.ts`.
* Cut from v1: Apple Photos placeholder card, `creative_jobs` table, end-user `AgentRunTimeline` (filed as candidates).
