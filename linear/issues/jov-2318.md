# JOV-2318: Releases: make /app/releases the canonical shell navigation route

- **Status:** Backlog
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2318/releases-make-appreleases-the-canonical-shell-navigation-route
- **Updated:** 2026-05-31T05:03:25.667Z

The app shell already supports /app/releases as the clean releases route, but several in-app navigation surfaces still send users to /app/dashboard/releases. Canonicalize user-facing navigation while preserving the dashboard path as an alias/backcompat route.

Acceptance:

* Dashboard sidebar releases nav, command palette route, keyboard shortcut, and canonical surface metadata use APP_ROUTES.RELEASES.
* Active-route matching continues to treat /app/dashboard/releases and /app/releases as the same shell route.
* Server redirects for unauthenticated releases direct to /app/releases.
* Tests cover nav/command/shortcut expectations for the canonical route.
* No production imports from exp routes and no visual redesign.
