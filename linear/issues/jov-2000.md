# JOV-2000: feat(agentos): extend AgentRunArtifact schema with model routing fields

- **Status:** Backlog
- **Priority:** 🟠 High
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2000/featagentos-extend-agentrunartifact-schema-with-model-routing-fields
- **Updated:** 2026-05-31T04:59:07.478Z

Extend the existing Zod schema in `apps/web/lib/agent-os/artifact.ts` with 9 new optional fields for model routing. Do NOT rename the schema or break existing consumers.

## Fields to add (all optional/nullable)

```ts
modelProvider?: string;       // 'anthropic' | 'openai' | 'google' | 'kilo' | 'openrouter'
modelName?: string;           // e.g. 'claude-sonnet-4-6', 'gpt-5.5'
routeReason?: string;         // why this model was chosen
attemptNumber?: number;       // retry count (starts at 1)
parentRunId?: string;         // UUID of parent run (for sub-agents)
candidateGroupId?: string;    // UUID grouping parallel candidates
validationScore?: number;     // 0-100 reviewer score
businessImpactScore?: number; // 0-100 estimated business impact
autonomyLevel?: 'supervised' | 'semi-autonomous' | 'autonomous';
```

Also extend `AgentBrief` type (when it ships from [JOV-1933](https://linear.app/jovie/issue/JOV-1933/build-agent-readable-issue-brief-generator)) with:

```ts
proposedModelRoute?: AgentModelRoute; // so Phase 3 router can read it
```

## Rules

* All 9 fields MUST be optional (`.optional()` in Zod) — never required
* Update `artifact.test.ts` test fixtures to include the new fields (or confirm they still pass without them)
* No breaking changes to existing `AGENT_RUN_MODEL_ROUTES` values

## Acceptance criteria

* Schema extended, types exported
* Existing tests pass without modification (new fields are optional)
* Typecheck passes
