## Goal

Track whether Jove's skill-generated assets are actually being saved and rated as high-quality by Tim within the 14-day window following PR1's merge to main. If the metric fails, pivot before adding more skills.

## Metric

* **Save rate:** ≥ 2 assets saved per week for 2 consecutive weeks (Tim's account).
* **Quality:** ≥ 7/10 average on a private internal rating log (Tim rates each saved asset 0-10 within \~24h of generation).
* **Pivot trigger:** save rate < 25% (saved / generated) over 14 days OR average quality < 5/10 over 14 days.

## Measurement

* Save rate auto-derives from `usage_events` table (created in PR1) — count `feature='jove.skill.save'` per week.
* Quality requires a private internal logging surface (admin-only `/admin/jove/quality-log` simple table; Tim enters scores manually). Build it as part of this issue, not PR1.
* Weekly Slack/email report to Tim summarizing the 14-day rolling window.

## Pivot path (documented; not auto-executed)

If pivot triggers: stop work on [JOV-2336](https://linear.app/jovie/issue/JOV-2336/jove-pr2-llm-proposer-3-more-skills-collab-birthday-viral-video) (PR2) before merge; spike a "Stanley-over-SMS"-style digest channel as an alternative surface; re-evaluate whether the chat-card UX is the right primary surface.
