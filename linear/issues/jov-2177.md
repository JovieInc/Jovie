## Symptom

All 145 `track()` calls in the codebase dispatch to `window.gtag` but no gtag/GTM/GA4 script is mounted anywhere. The entire conversion funnel is unobservable. `identify()` is a no-op. `Statsig.logEvent` is never called anywhere (zero exposure logging). Every PR that "added analytics for X" has been adding dead code.

Confirmed via live network capture against [staging.jov.ie](<http://staging.jov.ie>) on 2026-05-13: only Vercel Analytics (page views), Sentry, and Instantly Pixel fire. No product analytics destination reaches anything.

## Repro

```bash
# Live network capture in /browse on jov.ie homepage, signin, pricing:
# Zero requests to google-analytics.com, googletagmanager.com, mixpanel, posthog, segment
# Only: _vercel/insights/view, leadsy.ai (Instantly), /api/px, remarketstats.com

# Code grep:
grep -r "googletagmanager\|gtag/js\|GA_MEASUREMENT" apps/web/app/layout.tsx
# Returns: zero matches
```

## Evidence

* `apps/web/lib/analytics.ts:38-48` — `track(event, props)` calls `analyticsWindow.gtag(...)`, but gtag is never loaded
* `apps/web/lib/analytics.ts:56-59` — `identify(userId, traits)` body is `void userId; void traits;`
* `apps/web/lib/flags/statsig.ts:107-180` — zero `logEvent` calls anywhere
* `apps/web/lib/analytics/runtime-aware.ts:40-87` — server `trackEvent` only writes Sentry breadcrumbs in dev or on `$exception*`
* `apps/web/lib/analytics.ts:50-54` — `page()` no-op
* `apps/web/components/features/auth/SsoCallbackHandler.tsx:130-158` — zero `track()` calls on OAuth callback
* `apps/web/components/organisms/CookieBannerSection.tsx` — zero `track()` calls on cookie banner
* `apps/web/app/waitlist/page.tsx`, `waitlist/invite/page.tsx` — zero `track()` calls on waitlist + invite

## Root cause

The analytics module was stubbed for a future destination wire-up that never happened. New code continues to be written against the stub (145 call sites) under the assumption that events are reaching somewhere. They aren't.

## Recommended fix

Pick a destination and wire it. Recommended path:

1. **Vercel Analytics** `track()` **API** if we want minimal-deps + same vendor as page views
2. **PostHog** if we want full product analytics + feature flag + session replay
3. **GA4 via GTM** if we want marketing attribution + downstream Google Ads integration

After destination picked: rewire `track()`, `identify()`, `page()`, server `trackEvent` to that destination. Add `Statsig.logEvent` calls for conversion events (signup, subscription, magic_moment_achieved, onboarding_completed).

## Acceptance criteria

- [ ] Analytics destination chosen and configured (Vercel/PostHog/GA4 + decision rationale)
- [ ] `track()` body wired to destination
- [ ] `identify()` wired to destination (with PII review — handle/email decision)
- [ ] `page()` wired or removed (Vercel auto-tracks pageviews; decide if custom properties matter)
- [ ] Server `trackEvent` wired to destination server SDK OR removed
- [ ] `Statsig.logEvent` called for: `signup_completed`, `subscription_success`, `onboarding_completed`, `magic_moment_achieved`
- [ ] Add `oauth_callback_started`/`completed`/`failure` events with `{provider, durationMs, errorType}`
- [ ] Add `cookie_banner_viewed`/`accepted`/`rejected`/`dismissed` events
- [ ] Add `waitlist_saved`/`invite_detected`/`invite_invalid`/`invite_expired` events
- [ ] Hero CTA `handle` PII gated behind consent (or redacted)
- [ ] Define `funnel_step` enum + attach to every funnel event
- [ ] Add `route`, `previous_route`, `device`, `viewport`, `browser`, `app_version`, `feature_flags` properties to base track payload
- [ ] **Regression test**: Playwright loads `/`, asserts at least one analytics network request to the chosen destination
- [ ] **Regression test**: Vitest asserts `identify()` body invokes destination SDK
- [ ] **Lint rule**: ban new `track()` calls that don't include required base properties
- [ ] PR includes screenshot of analytics dashboard receiving events

## Regression proofing

* **Test**: Playwright E2E asserts destination receives at least one event per page load
* **Test**: Vitest unit test asserts `identify()` and `Statsig.logEvent` are not no-ops
* **Lint / hook**: Custom ESLint rule requires `track()` calls to include `funnel_step` for funnel events
* **Observability**: Sentry alert if `track()` calls > N/min and destination receives 0 events (canary)
