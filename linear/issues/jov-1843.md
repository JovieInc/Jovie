## Wave C2 — Artist-facing campaign builder (v1)

Surface for artists to spin up a tracked alert-growth campaign without leaving Jovie. Resist the urge to build ad-tech: this is a thin layer over `audienceSourceGroups`/`audienceSourceLinks`.

## Scope

### Route

* `apps/web/app/app/(shell)/dashboard/alert-growth/page.tsx`

### Features (v1, deliberately small)

* Create draft campaign (name, optional budget, destination = landing page from Wave B1).
* Auto-generate trackable landing URL (writes `audience_source_groups` + `audience_source_links`).
* Copy URL / copy CTA assets (text/image kit) for paste into Meta/IG/etc.
* Stats panel: clicks, intent creations, verified subscribers, conversion rate.
* Per-source breakdown using existing retargeting attribution endpoint.

### NOT in v1

* Ad spend execution.
* Multi-platform pixel management.
* A/B asset generation.
* A separate `alert_growth_campaigns` table — defer until source-group metadata proves insufficient.

## Acceptance criteria

- [ ] Artist can create + name a campaign and get a copyable verified URL.
- [ ] Stats reflect real conversion data within current cron cadence.
- [ ] Page follows shell-v1 surface rules (`DashboardWorkspacePanel` + bordered content container).
- [ ] No duplicate page title (breadcrumb owns it).
- [ ] Screenshot in PR body.

## Dependency

Wave B1 (landing page) must exist for the campaign to point at something.
