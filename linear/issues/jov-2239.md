## Follow-up

v1 has a single hardcoded `extractEventSignal` function. Once two signal patterns exist (events + releases via ReleaseOpsAgent), generalize: `InboxSignalAgent` reads `external_objects.kind='gmail_message'` and emits multiple `context_facts` kinds in one pass (event_signal, release_signal, payment_signal, contract_signal, etc.). Reduces Gateway-token cost vs. running multiple specialized extractors.

## Why it matters

Cost-of-AI scales with number of agent passes per email. Consolidating into one extractor with a richer Zod-validated output schema is \~3× cheaper than three separate calls. But the prompt becomes harder to iterate. Trade-off worth re-evaluating after eval.
