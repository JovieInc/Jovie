## Goal

Tim can open chat, tap "Generate announcement carousel," and within \~10–20s see a Top Move card with a real generated photo carousel tied to a real upcoming release. Save → asset persists in R2 and shows in calendar. Tweak → quote-and-reply in chat composer (regeneration ships PR2).

## Scope (in PR1)

* Calendar added to primary nav. Tasks STAYS in nav (Tasks-out-of-nav has orphan-flow risk; deferred to separate issue).
* New durable async job: `/api/jove/run-skill` enqueues into existing `ingestionJobs`, returns ids immediately. Worker calls AI Gateway (Grok Imagen), uploads to R2, transitions `suggestedActions.status` via CAS.
* One hardcoded skill end-to-end: `photo.carousel.release-announce` with Tim-authored prompt template.
* Typed hardcoded skill registry at `apps/web/lib/jove/skills/index.ts`. No filesystem auto-discovery (broken on Vercel serverless).
* New R2 image buckets: `jovie-jove-assets-{dev,stg,prod}` + `jovie-jove-assets-permanent-{dev,stg,prod}`. 30-day TTL on transient; promote-on-save to permanent.
* CSP entries for the new bucket domains in `apps/web/constants/platforms/cdn-domains.ts`.
* Additive migration: extend `suggested_action_status_enum` with `running`, `ready`, `failed`, `saved`, `dismissed`. Add composite index `(user_id, kind, created_at)`.
* TopMoveCard component (single primary card; MoreIdeasDisclosure collapsed default; mobile-first; ≥44px touch targets; keyboard nav; optimistic Save with 5s undo).
* JoveThinking progress component wired to job state.
* Tweak v1 = quote-and-reply in chat composer.
* Calendar renders saved Jove assets with Jove glyph + deep link to originating chat message.
* Single canonical usage source of truth: explore `apps/web/lib/entitlements/`, `lib/notifications/`, billing first; extend if found, else create `usage_events` append-only table.
* Per-tier quotas: Free (1 lifetime), Pro $39 (\~10/mo), Max $149 (\~50/mo). Modeled at 85-90% gross margin on gen cost. Quota check BEFORE AI Gateway call. Surface remaining quota in chat header chip.
* Pro+ entitlement gate on `/api/jove/run-skill`.
* Structured prompt data-block utility (no prompt injection from upstream titles/names). Mirrors `apps/web/lib/connectors/gmail/extract-event-signal.ts` pattern.
* Tests: registry, context-builder (≤3 db.select asserted via spy), run-skill-worker (every failure path), prompt-data-block (hostile fixture suite), asset-hosting (owner-scoped signed URLs), API routes (entitlement + cross-user + double-click idempotency + CAS conflict), TopMoveCard (every state + optimistic save reversion).
* Playwright E2E at `apps/web/tests/e2e/jove-skill-run.spec.ts`: click Generate → job pending → job ready → save → calendar shows entry.
