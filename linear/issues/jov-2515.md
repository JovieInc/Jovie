# JOV-2515: fix(a11y): cookie banner buttons below 44px touch target minimum

## Source

* Current issue: ad-hoc (design-review audit 2026-05-21)

* Source branch/session: cos/design-review-perf (orchestration)

## Follow-up

Cookie banner buttons (Reject, Customize, Accept All) have height 26px — well below the 44px minimum touch target. Nav "Sign in" link is 28px tall. Primary hero "Request access" CTA in header is 32px tall. All interactive elements visible on every public-facing page.

Found on: [https://staging.jov.ie](<https://staging.jov.ie>) (and all pages — cookie banner is persistent)

Specific elements:

* Reject: 51×26px
* Customize: 74×26px
* Accept All: 74×26px
* "Sign in" nav: 62×28px
* "Request access" header CTA: 129×32px

## Why it matters

These are the first interactive elements most users encounter (before dismissing the cookie banner). On mobile, 26px touch targets cause mis-taps and user frustration. WCAG 2.5.5 recommends 44×44px for all targets.

## Classification

Required

## Acceptance criteria

* All cookie banner buttons: min-height 44px
* Nav "Sign in" link: min-height 44px (padding increase)
* Header "Request access" CTA: min-height 44px (currently 32px)
* Verified via touch target audit: `[...document.querySelectorAll('a,button')].filter(e => {const r=e.getBoundingClientRect(); return r.width>0 && r.height<44})`
