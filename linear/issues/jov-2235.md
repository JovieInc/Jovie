## Follow-up

The AI_CONNECTORS_BETA flag exposes 5–10 design-partner DJs to recurring Gateway-token cost (extractor runs on cron + on-demand). Per `.claude/rules/infra.md` and `docs/COST_MONITORING.md`, recurring AI work requires a Cost Impact disclosure AND an auto-rollback hook that kills the flag if spend exceeds a threshold.

Implement the hook so that if daily Gateway spend for users gated by `AI_CONNECTORS_BETA` exceeds `AI_CONNECTORS_DAILY_TOKEN_BUDGET * activeBetaUserCount * 1.5`, the flag automatically flips to disabled and Tim gets a Sentry breadcrumb + Slack ping.

## Why it matters

Without this, an extractor regression that triples token usage doesn't trip an alarm until the monthly Gateway bill arrives. Closed-beta is exactly when this hook needs to be live.

## Classification

Required

## Acceptance criteria

* Hook runs hourly via existing `process-ingestion-jobs` cron.
* Computes 24h Gateway spend attributable to `AI_CONNECTORS_BETA`-gated users (`agent_runs.cost` aggregate over user IDs in the allowlist).
* On threshold breach: Statsig API call flips `AI_CONNECTORS_BETA` to disabled, posts Sentry warning, posts Slack notification.
* Test asserts threshold computation + flag-flip mock.
