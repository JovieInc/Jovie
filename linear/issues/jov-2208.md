# JOV-2208: [P1][Demo] Build campaign recommendation engine with timing and revenue math

- **Status:** Todo
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex

## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo recommends a 72-hour The Deep End Weekend Drop with product options and revenue math.
* Expected behavior: Jovie should turn opportunity context, catalog fit, audience size, product assumptions, and timing into ranked campaign recommendations.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

This is the bridge between insight and action. Without recommendation logic, the product cannot justify why this campaign, why now, and what success should mean.

## Likely root cause

Campaign math currently lives only in demo copy. There is no production recommendation model or auditable assumptions layer.

## Acceptance criteria

* Recommendations include why now, target audience, channels, expected order/revenue path, assumptions, and confidence.\\n- Multiple campaign options can be compared.\\n- Revenue math is deterministic and testable.\\n- Recommendations can be accepted, edited, or dismissed.

## Test plan

* Unit tests for revenue math and ranking.\\n- Fixture test for tee/poster/hoodie recommendations.\\n- Snapshot/API contract tests for chat-renderable recommendation payload.\\n- Manual route check with stable UI and no duplicate CTAs.

## Suggested fix

Build a recommendation service that accepts an opportunity plus artist graph context and returns typed campaign option payloads for chat and campaign surfaces.
