## Follow-up

[JOV-1581](https://linear.app/jovie/issue/JOV-1581/artist-profile-full-email-signature-generator-with-jovie-branding)'s spec called for an inline PNG icon row for socials (Instagram | Spotify | Apple Music | …) — CDN-hosted PNGs only, no SVG, no base64. The v1 implementation in `apps/web/lib/email-signature/build-signature.ts` ships text labels separated by `·` instead (functional and email-client safe, but less visually polished).

To upgrade: pick a brand-icon set (e.g. simple-icons rendered to PNG, or a curated Jovie set), host the PNGs at known stable `https://` URLs, and map each social `platform` → icon URL inside `build-signature.ts`. Honor the existing `isSafeHttpsUrl` gating and keep table-only layout.

## Why it matters

The icon row is what makes the signature feel premium and brand-aligned with Jovie's profile UI. Text-only socials are acceptable but read as a fallback, not a finished design.

## Classification

Candidate

## Acceptance criteria or triage question

Triage question: where do we host the PNG icon set (Vercel public/, a dedicated CDN bucket, or a third-party brand set)? If we don't have a stable host for branded PNG icons, close this — text labels are deliberately safer for spam scoring until we own the asset path.
