# JOV-2519: fix(start-page): add scaffolding/prompt chips to chat intake for first-time visitors

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex

- **Updated:** 2026-05-30T23:17:47.911Z

## Source

* Source: /design-review audit of [staging.jov.ie](<http://staging.jov.ie>) (2026-05-21)
* Session: cos/design-review-perf orchestration run

## Follow-up

The `/start` and `/waitlist` pre-auth chat intake surfaces show "Hey, I'm Jovie. / What are you working on?" over an empty 720px dark canvas with no context or scaffolding. First-time visitors arrive from the marketing homepage with no framing of what to type.

DESIGN.md note: anthropomorphic self-introduction is borderline acceptable on this surface (first interaction), but the empty canvas without any suggested prompts or context is cold and creates friction.

## Why it matters

Conversion: A blank-canvas chat box with no scaffolding creates hesitation for users who don't know what Jovie does. Adding 3 suggested prompt chips ("Distribute my next release", "Pitch my music to a playlist", "Plan a tour") immediately communicates the product's value and reduces time-to-first-interaction.

## Classification

Required

## Acceptance criteria

* `/start` shows 3-4 suggested prompt chips below the input area
* Chips use the existing pill/badge component from the design system
* Clicking a chip pre-populates the input (does not auto-submit)
* Chips are relevant to the creator use case (not generic)
* No layout shift when chips are hidden after user starts typing
* Mobile-optimized (chips stack or scroll horizontally on small screens)

## Dependency

None — standalone UI addition to the start/waitlist page
