## Problem

The `/sign-up` page on [staging.jov.ie](<http://staging.jov.ie>) contains two `<h1>` elements. Each page should have exactly one h1 — having multiple breaks screen-reader heading hierarchy and fails axe rule `page-has-heading-one` / `landmark-no-duplicate-main`.

## Suspected source

Likely two h1s rendering at once:

1. The auth shell brand panel headline ("Built for Artists." or similar)
2. The Clerk-rendered SignUp component's internal heading

The brand panel headline in `apps/web/components/auth/...` should be h2 (or aria-label only, not a heading), while Clerk's component header remains the page's primary h1.

## Acceptance criteria

- [ ] `/sign-up` and `/signin` each render exactly one `<h1>`
- [ ] axe-core run on both pages reports zero `page-has-heading-one` and `heading-order` violations
- [ ] Existing `apps/web/tests/e2e/a11y/*` covers this (or add a new test)

## Repro

```bash
$B goto https://staging.jov.ie/sign-up
$B js "document.querySelectorAll('h1').length"
# Expected: 1, Actual: 2
```
