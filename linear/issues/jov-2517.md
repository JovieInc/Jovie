## Follow-up

The `/start` (waitlist/chat intake) page shows a dark canvas with "Hey, I'm Jovie. / What are you working on?" and an empty chat input. There is no scaffolding for a first-time visitor — no suggested prompts, no context about what Jovie can help with, no example queries.

The goodwill reservoir assessment: the blank dark canvas with an open-ended question creates cognitive load. The user has to invent a use case with zero orientation. This slows time-to-value and increases bounce rate from waitlist landing.

## Why it matters

This is the first product experience for waitlisted users. A cold start with a blank canvas and "What are you working on?" is a conversion risk — the user has to already know what to ask before they can engage.

## Classification

Candidate

Pickup agent must first judge whether to implement, close, or split this.

## Acceptance criteria or triage question

Should the chat intake show 3-4 suggested prompt chips below the input (e.g., "Help me plan my next release", "What should I focus on this week?", "Check my streaming performance")? Or is the blank canvas intentional for the creator persona who already knows exactly what they want?
