## Follow-up

CEO review noted that Google Calendar dominance in music ops is real but not absolute — DJs/managers heavily use iCloud Calendar too. v1 ships Google Calendar only because the persona (touring DJ 40+ shows/yr) is Google-dominant in the design-partner cohort. Pick up Apple iCloud Calendar when beta feedback explicitly requests it.

When picking up: iCloud uses CalDAV with app-specific passwords (no OAuth). Token storage and refresh behavior differ — `ConnectorDriver` interface should already support this via `getAuthorizeUrl` returning a "configure CalDAV" UI path instead of an OAuth redirect.

## Why it matters

Forcing iCloud users to Google Calendar to use Jovie's magic moment is a switching cost they won't pay. Once Google is proven, iCloud unlocks the Apple-leaning DJ segment.

## Classification

Candidate

Pickup agent must first judge whether to implement, close, or split this. Trigger: ≥20% of beta-cohort DJs use iCloud as their primary calendar.

## Dependency

Depends on H-PR-1 (ConnectorDriver interface formalized).
