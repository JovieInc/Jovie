## Follow-up

`pnpm --filter web lint:eslint` crashes when ESLint 10.2.1 scans `.mjs` files in `apps/web/`. The crash occurs in:

* `apps/web/scripts/*.mjs` (e.g., `analyze-issues.mjs`, `approve-admin-production.mjs`, `approve-waitlist-entry.mjs`)
* `apps/web/stories/` directory
* Possibly other `.mjs` files in `apps/web/lib/`

The error is:

```
TypeError: scopeManager.addGlobals is not a function
  at addDeclaredGlobals (.../eslint/lib/languages/js/source-code/source-code.js:221:15)
  at SourceCode.finalize (.../eslint/lib/languages/js/source-code/source-code.js:1090:3)
```

`apps/web/eslint.config.js` has no special handling for `.mjs` files — the same rule set (including custom `@jovie/*` plugins) is applied, and one of those plugins passes an incompatible scope manager API at ESLint 10.2.1.

Narrowing: `.ts` files lint cleanly; the crash is `.mjs`-specific. Bisecting the config to identify the exact plugin/rule is the fix path.

## Why it matters

`pnpm --filter web lint:eslint` is a CI gate. When it crashes on `.mjs` files, lint coverage for all those files is zero — silent coverage gap on scripts and stories that touch production-adjacent logic.

## Classification

Required

## Acceptance criteria

* CI lint step passes cleanly on a branch that includes a `scripts/*.mjs` file
* Root cause documented (which plugin/rule triggers the incompatible API call)
