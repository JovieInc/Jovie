# JOV-2564: Fix broad web test typecheck failures

- **Status:** Todo
- **Priority:** 🟡 Medium
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** grok
- **URL:** https://linear.app/jovie/issue/JOV-2564/fix-broad-web-test-typecheck-failures
- **Updated:** 2026-05-31T05:04:25.756Z

While shipping [JOV-2561](https://linear.app/jovie/issue/JOV-2561/add-promptfoo-baseline-eval-suite-for-jovie-chat), `pnpm --filter @jovie/web run typecheck:tests` failed on existing broad test/script typing errors outside the Promptfoo eval suite.

Observed top failures include:

* `scripts/drizzle-migrate.ts(213,24)` implicit `any` for `error`.
* `scripts/drizzle-seed.ts(1434,45)` seed contact insert type mismatch around `deviceType`.
* `scripts/generate-brand-assets.ts(265,26)` missing `previewBackground` on asset config union.
* `scripts/overnight-qa/server.ts` readonly/nullability process handling errors.
* `tests/unit/dashboard/*`, `tests/unit/profile/*`, and release/provider fixture type drift including missing `status`, `shareContext`, `eventType`, and `confirmationStatus` fields.

The focused check `pnpm --filter @jovie/web exec tsc -p tsconfig.test.json --noEmit --incremental false --pretty false 2>&1 | rg 'tests/eval/promptfoo|jovie-chat-provider|assertions\.cjs|promptfooconfig' || true` returned no Promptfoo eval-suite errors after the [JOV-2561](https://linear.app/jovie/issue/JOV-2561/add-promptfoo-baseline-eval-suite-for-jovie-chat) changes.

Keep this issue scoped to restoring the broad test typecheck gate; do not fold it into the Promptfoo baseline PR.
