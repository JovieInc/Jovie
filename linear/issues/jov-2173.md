## Unaddressed CodeRabbit findings to triage

### Major

1. **DB uniqueness key weaker than app dedupe key** (`apps/web/drizzle/migrations/0046_social_links_active_url_unique.sql`). The partial unique index uses `(creator_profile_id, platform, lower(url))`, but the application-layer `dedupeKey` normalizes more aggressively (strips `www.`, trailing slash, casing on the whole URL). Result: the DB lets through near-duplicates that the app would collapse for display. Acceptable as defense-in-depth, but worth aligning. Consider a generated column or a stricter normalization function in the index.
2. **Dedupe drops URL query components** (`apps/web/lib/utils/social-platform.ts` `dedupeKey`). For most social platforms query params are tracking noise and should be stripped — but some legitimate URLs differ only in query (e.g. YouTube playlist IDs). Audit which platforms need query-preserving dedupe and either platform-scope the strip or use a allowlist.

### Refactor suggestion

3. **Use Drizzle ORM query composition** (`apps/web/scripts/audit-duplicate-social-links.ts`). The script uses raw SQL templates; Drizzle's composition API would be safer and easier to maintain.

### Minor

4. `scanForBrokenStrings` **only captures first match per pattern** (`apps/web/tests/helpers/broken-string-scan.ts:73`). Currently uses `pattern.exec(text)` once per pattern; for diagnosability, switch to global match (`text.matchAll(pattern)` with `/g` flag) so all instances surface.
5. **Fail fast on invalid stability config** (`apps/web/tests/helpers/dom-stability.ts:68`). The helper accepts both `selector` and `absentSelector` as optional but if neither is provided it polls silently and always passes. Throw at the top of `assertDomStable` when both are absent.

### Nitpick

6. **Consider concurrent index creation** (`apps/web/drizzle/migrations/0046_social_links_active_url_unique.sql`). `CREATE UNIQUE INDEX` blocks writes; `CREATE UNIQUE INDEX CONCURRENTLY` does not. Social_links is hot. Migration runner would need to support non-transactional execution for the CONCURRENTLY branch.
