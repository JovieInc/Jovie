# JOV-2201: [P1][Performance] Dashboard routes repeat shell requests and abort core API calls during navigation

- **Status:** Done
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2201/p1performance-dashboard-routes-repeat-shell-requests-and-abort-core
- **Updated:** 2026-05-31T05:05:54.784Z

## Evidence

* Screenshot/frame: /Users/timwhite/.codex/worktrees/465c/jovie-v1/apps/web/.gstack/qa-reports/demo-readiness-2026-05-14-live/screenshots/04-dashboard-releases.png
* Screenshot/frame: /Users/timwhite/.codex/worktrees/465c/jovie-v1/apps/web/.gstack/qa-reports/demo-readiness-2026-05-14-live/screenshots/07-tasks.png
* Route: /app/dashboard/releases and /app/dashboard/tasks
* Observed behavior: Live audit measured duplicate requests on dashboard routes. Releases requested /api/version 4 times, /api/billing/status 2 times, and /api/chat/conversations 2 times. Tasks requested /api/version 4 times, POSTed /app/dashboard/tasks 3 times, and duplicated billing/chat requests. Several billing/chat requests ended as net::ERR_ABORTED during route changes.
* Expected behavior: Shell-level data should be cached/deduped, and route transitions should avoid duplicated core shell fetches and aborted user-visible requests.

## Impact

This contributes to sluggish route transitions and makes loading/flicker harder to stabilize. It also adds noise to console/network QA.

## Likely root cause

Shell providers and route components mount overlapping query/fetch paths. Some shell data appears to fetch outside a shared TanStack Query key or without sufficient stale time/deduping.

Likely files/components:

* apps/web/lib/version/use-web-update.ts
* apps/web/lib/queries/useChatConversationsQuery.ts
* apps/web/lib/queries/useBillingStatusQuery.ts or billing status consumer
* apps/web/app/app/(shell)/dashboard/tasks/task-actions.ts
* apps/web/app/app/(shell)/dashboard/releases/page.tsx

## Acceptance criteria

* /api/version is requested at most once per route mount window.
* Billing/chat shell requests are deduped across route transitions.
* No net::ERR_ABORTED for core shell requests during normal navigation.
* Route transition time improves or stays stable with no additional flicker.

## Test plan

* Re-run the Playwright network audit.
* Add a targeted e2e/network assertion for duplicate /api/version and billing/chat requests.
* Verify app shell remains mounted across releases/tasks navigation.

## Suggested fix

Move repeated shell fetches behind stable TanStack Query keys or a singleton provider-level cache, then keep cached data visible while refetching instead of remounting local fetches per route.
