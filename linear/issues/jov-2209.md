## Impact

The operator promise depends on moving from recommendation into concrete commerce setup instead of leaving the artist with a task list.

## Likely root cause

Merch/drop creation is not yet a cohesive production workflow connected to campaigns, smart links, and fan-facing surfaces.

## Acceptance criteria

* Drop records include product, price, inventory/fulfillment settings, artwork/assets, launch window, and owner.\\n- Draft drops can be previewed before publish.\\n- Drop state is visible in campaign status.\\n- Product-page creation is idempotent.

## Test plan

* Unit tests for drop state transitions.\\n- Integration test for recommendation to draft drop.\\n- Manual preview of fan-facing product page.\\n- Payment/fulfillment boundaries reviewed before enabling real checkout.

## Suggested fix

Add a merch/drop workflow domain with draft-first creation, typed campaign linkage, and fan-facing preview surfaces before any live payment behavior.
