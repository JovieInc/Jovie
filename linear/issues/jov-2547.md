## Source

* Current issue: ad-hoc Codex request
* Source branch/session: codex/public-profile-system-b-hardening

## Scope

Harden the highest-traffic public profile surfaces to feel native-iOS and System B aligned, using existing Jovie design tokens and without rewriting System B rules.

## Acceptance criteria

* /tim public profile Home, Music, Events, Alerts, More, and email signup flow use compact System B scale comparable to Instagram/TikTok profile entry points.
* Public profile copy removes obvious duplication and low-signal helper text.
* Public profile states avoid horizontal overflow, clipping, image distortion, bottom-nav overlap, and layout shift on phone widths.
* Focused regression coverage is added or updated for the touched profile surfaces and signup flow.
* QA evidence includes mobile-first screenshots and relevant typecheck/lint/test results.

## Notes

Homepage is intentionally out of scope except for shared-token regressions, per request.
