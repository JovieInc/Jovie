# JOV-2103: design(admin/settings): move operational controls to Ops control panel (JOV-2097)

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** —
- **URL:** https://linear.app/jovie/issue/JOV-2103/designadminsettings-move-operational-controls-to-ops-control-panel-jov
- **Updated:** 2026-05-31T02:16:20.932Z

## Problem

Operational toggles (waitlist, maintenance mode, invite gates, etc.) currently live in `/admin/settings`. Settings should be for persistent configuration (profile, branding, integrations), not for live operational controls that an admin flips during incidents or launches.

Having operational controls in Settings forces admins to navigate away from the Ops screen during time-sensitive situations.

## Required

Once [JOV-2097](https://linear.app/jovie/issue/JOV-2097/designadminops-add-control-panel-card-for-operational-toggles) (Ops control panel card) lands:

1. Identify all operational toggles currently in Settings
2. Move them to the Ops control panel ([JOV-2097](https://linear.app/jovie/issue/JOV-2097/designadminops-add-control-panel-card-for-operational-toggles))
3. Remove the corresponding Settings sections
4. Add a redirect or tooltip in Settings pointing admins to Ops for operational controls if the UX requires it

## Blocked by

[JOV-2097](https://linear.app/jovie/issue/JOV-2097/designadminops-add-control-panel-card-for-operational-toggles) — Ops control panel card must exist before controls can be moved there

## Acceptance criteria

* No operational on/off toggles remain in Settings after this lands
* All moved controls work correctly in their new location
* Settings page is shorter and focused on configuration
