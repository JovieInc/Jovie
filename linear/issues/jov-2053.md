## Follow-up

The Apple Developer account currently in use for Mac code signing and notarization is registered as an **individual** in Tim White's name. This is reflected in the active Developer ID Application certificate:

```
Subject: Developer ID Application: Tim White (G24T327LXT)
```

That means Gatekeeper / first-launch dialogs, the "About → Signed by" panel in macOS, and any system-level signer attribution will display **"Tim White"** rather than **"Jovie Inc."**

This is acceptable for now (and ships the app today), but it will look unpolished once we have funded company optics, enterprise customers, or press writeups screenshotting macOS dialogs.

## Why it matters

* Brand polish: macOS surfaces show the signer name. Investors / press / first impressions notice.
* Enterprise / IT-procurement flows sometimes inspect the certificate org field.
* The Team ID stays the same on conversion (`G24T327LXT`), so existing app installs continue to receive auto-updates after the conversion.
* Conversion to a corporate Apple Developer account requires a **D-U-N-S number** for the legal entity (Jovie Technology Inc.) and the \~$99/year Apple Developer Program membership transferred / re-registered.

## Classification

Candidate follow-up: Pickup agent must first judge whether to implement, close, or split this.

## Triage question

Has Jovie Technology Inc. closed its next funding round and is ready to take on the operational steps for the conversion? If yes, proceed with the steps below. If no, leave open.

## Decision rule

**Do not convert until after the next funding round closes.** Conversion has a few-day Apple review window plus paperwork, and there's no operational benefit until we're routinely showing the brand to investors / customers in macOS dialogs.

## Steps when ready

1. Obtain a D-U-N-S number for Jovie Technology Inc. via [dnb.com](<https://www.dnb.com/>) (free, 1–5 business days).
2. Apple Developer Program → Account → Membership → request company / organization conversion. Apple verifies the D-U-N-S, the legal entity name match, and that the requester has signing authority (Tim, as founder).
3. After approval, generate a **new** Developer ID Application certificate from the same Team ID (now company-attributed). Old cert remains valid until expiry; new cert takes over.
4. Re-export the new `.p12`, update `CSC_LINK` and `CSC_KEY_PASSWORD` in:
   * Doppler `prd` and `stg` (`jovie-web` project)
   * GitHub Secrets on `JovieInc/Jovie`
5. Trigger one staging build to verify the new signer attribution shows "Jovie Inc." instead of "Tim White".
6. Trigger a production release. Existing installed apps continue auto-updating because `Team ID` is unchanged.
7. (Optional) Revoke the individual cert at [developer.apple.com](<http://developer.apple.com>) after the new cert is in production for ≥ 1 release cycle.
