Founder command center for approvals, deploys, runway, YC signals, and priorities. Every ops workflow eventually surfaces in `/admin/ops`. Parent epic for the operational HUD system.

See [JOV-1846](https://linear.app/jovie/issue/JOV-1846/feat-operational-hud-ai-driven-ops-dashboard-for-founder) for the original design spec: `/admin/ops`, personal priorities, approval queue, AI priority assessment, YC signals, subscriptions, content orchestration, knowledge statements, bottlenecks, and core values.

**Product output:** Tim can see survival, YC, deploy, and approval state from one screen without manually querying 5+ systems.
**Velocity output:** HUD data contract used by future crons; one approval manifest shape for all agents.
**Owner profile:** Chief / founder-os / coder
**Cost route:** deterministic (read-only seeded data); economy for refresh crons
**GStack skill(s):** /qa, /review, /ship
**Verification:** /admin/ops renders all sections; approval queue items have decision, risk, owner, expiry, proposed action, and why-now; daily cron runs with no premium model calls
**HUD surface:** /admin/ops panel
**Human approval required:** YES (deploy panel reads CI state; approval queue is the Tim-gate for all agents)
