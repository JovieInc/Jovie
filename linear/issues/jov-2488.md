# JOV-2488: Canonicalize tasks shell rows and data loading

- **Status:** Done
- **Priority:** 🟠 High
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** category:app-shell, parallelizable, performance, ui, design-system, type:chore, area:ui, qa:e2e, area:app-router, type:test, ux
- **URL:** https://linear.app/jovie/issue/JOV-2488/canonicalize-tasks-shell-rows-and-data-loading
- **Updated:** 2026-05-31T03:48:42.661Z

Unify `/app/dashboard/tasks` around the shared shell row/list/table primitives and route data contract.

Scope:

* Keep current task behavior and APIs; do not fake unavailable task actions.
* Normalize task row spacing, labels, menus, empty/loading/error states, and keyboard/accessibility behavior to the shell token system.
* Preserve App Router boundaries: route data in the page/leaf adapter, persistent chrome in shell layout.
* Add focused tests for row rendering, task ordering/state labels, and route shell persistence where practical.
* Run focused typecheck, Biome, unit/component tests, and screenshot/design review for desktop and mobile if visual output changes.

Acceptance:

* Tasks page uses canonical shell primitives and no one-off row/list styling drift.
* No `fallback={null}` shell pop-in around task rows.
* Existing task APIs and command/menu behavior keep working.
