## Follow-up

PR #8637 fixes the 500 by wrapping getOrMintOnboardingSessionId() in try/catch + redirect. But /start will redirect to /signup on every hit until SESSION_SECRET is provisioned in Doppler jovie-web/prd (and /stg). Once the secret is live, the route works end-to-end.

Steps:

1. doppler secrets set SESSION_SECRET --project jovie-web --config prd (generate a 32+ char random secret)
2. doppler secrets set SESSION_SECRET --project jovie-web --config stg
3. Redeploy (or wait for next deploy) so the new env is picked up
4. curl -s -o /dev/null -w "%{http_code}\\n" [https://jov.ie/start](<https://jov.ie/start>) should return 200 (not 302 or 500)

## Why it matters

/start is the front door for anonymous onboarding (JOV-2132). Without SESSION_SECRET, every landing redirects to /signup, skipping the onboarding chat flow entirely.

## Classification

Required

## Acceptance criteria

curl [https://jov.ie/start](<https://jov.ie/start>) returns 200 and renders OnboardingShell. Sentry no longer fires the SESSION_SECRET captureError for /start requests.
