# JOV-2096: design(admin/ops): remove redundant status badges from Kanban cards

## Problem

Kanban cards in `/admin/ops` display status badges (e.g. "In Progress", "Backlog") inside the card body. This is redundant — the column the card is in already communicates its status.

The badges add visual noise without informational value and make the board feel like a generic AI-generated admin template.

## Expected

* Remove status badges from Kanban card bodies
* Cards display only: primary label + essential metadata (assignee, due date, priority indicator if any)
* Status remains readable from column headers alone
