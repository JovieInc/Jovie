## Follow-up

The /\[username\] profile route mounts 4 independent client components directly in page.tsx: PublicClaimBanner, ErrorBanner, DesktopQrOverlayClient, ProfileViewTracker, and JoviePixel. Each creates its own chunk boundary, multiplying the shared vendor chunk count. Wrapping these in a single ProfileClientGroup client component would reduce the number of independent client module entries to 1.

## Why it matters

Estimated reduction: 6-8 initial chunks. This is the lowest-risk path toward the JOV-2271 ≤30 chunk target. No functional change — same components rendered, just grouped under one client boundary.

## Classification

Candidate

## Acceptance criteria or triage question

Pickup agent must first judge whether to implement, close, or split this.

If implementing: verify SSR of each component is preserved after grouping, build, confirm initial chunk count drops by ≥4 vs baseline (49 chunks from JOV-2271 measurement).

## Dependency

Depends on JOV-2271 investigation landing (PR #9190) for baseline method.
