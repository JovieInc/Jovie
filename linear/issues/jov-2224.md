# JOV-2224: Unified app shell migration: Library slice

- **Status:** Done
- **Priority:** 🟠 High
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** Unified App Shell Migration
- **Labels:** area:core-flows, area:app-router, polish, type:feature, design-system, category:app-shell, parallelizable, ux, ui
- **URL:** https://linear.app/jovie/issue/JOV-2224/unified-app-shell-migration-library-slice
- **Updated:** 2026-05-31T03:48:39.764Z

## Scope

Port the production Library route into the unified app shell with the best shell-v1 improvements while preserving production data contracts and App Router shell persistence.

## Contracts

* Artist/operator job: browse, search, sort, and act on real library/releases without losing shell chrome or audio/sidebar state.
* Primary action: inspect a release/asset from library grid/list and navigate via sidebar/library context.
* Modes: real-data grid and list modes, search/sort/static presets, facet/sidebar context, asset drawer, empty/loading/error states.
* Responsive: dense desktop grid/list, usable tablet, mobile without horizontal overflow.
* Performance: no production imports from apps/web/app/exp/\*; avoid shell remount, avoid route-level waterfalls, keep loaders minimal.
* Keep/adapt/delete: adapt only through production contracts such as LibraryReleaseAsset and existing release/library route types; no fake persistence, no favorites, no fake edit flows.

## Verification

* Focused unit/component coverage for library mapping/adapters and mode state where applicable.
* E2E shell persistence across /app/library and neighboring shell routes.
* Desktop/tablet/mobile screenshots and pixel diff against current production and /exp/shell-v1 where useful.
* Run scoped design review and QA before approval/ship.
