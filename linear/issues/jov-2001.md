# JOV-2001: feat(agentos): add ModelRoutePolicy type + route registry

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned

- **Project:** —

Add three new model routes to `AGENT_RUN_MODEL_ROUTES` in `apps/web/lib/agent-os/artifact.ts` and define a `ModelRoutePolicy` type.

## New routes

* `kilo-free` — Kilo SDK, max 5 parallel, ≤100 calls/hour, fallback: openrouter-cheap
* `openrouter-cheap` — OpenRouter free tier, estimated $0.01/call, fallback: ai-sdk-gateway
* `airme-agent` — Airme SDK, internal-only, for cost-sensitive tasks

## ModelRoutePolicy type

```ts
interface ModelRoutePolicy {
  routeKey: AgentModelRoute;
  provider: string;
  modelId: string;
  maxParallel?: number;
  callsPerHour?: number;
  estimatedCostPerCall?: number;
  fallbackRoute?: AgentModelRoute;
  safeTaskTypes?: string[]; // restrict to these task types if set
}
```

## Documentation

These routes must be documented in `agentos/memory/model-routing.md` (see Phase 5 seeding issue) so agents can understand what they mean.

## Acceptance criteria

* 3 new route keys added to `AGENT_RUN_MODEL_ROUTES` union
* `ModelRoutePolicy` type exported
* Policy registry map exported (routeKey → policy)
* Typecheck passes
