## What this PR does (PR-1 of 2)

Establishes the unified `AdminPage` shell pattern and rewrites the Overview page (`/app/admin`) to lead with hero metrics. Lays the foundation that PR-2 (settings consolidation + remaining 4 page migrations) builds on.

### Scope

* New `apps/web/components/features/admin/layout/AdminPage.tsx` shell — flat header, optional hero slot, optional tabs slot. `space-y-6` outer rhythm matching dashboard canon.
* Extend `AnalyticsCard` (`apps/web/components/features/dashboard/atoms/AnalyticsCard.tsx`) with `variant="hero"` — no card wrapper, `text-[36px] font-bold tracking-[-0.03em]`. Used by both admin and future dashboard heroes.
* Extend `WorkspaceTabsSurface` with `headerless` prop so admin pages with both hero + tabs don't stack 3 competing headers.
* Rewrite `AdminToolPage` and `AdminWorkspacePage` as `@deprecated` shims re-exporting `AdminPage`. **Do not delete** — 24 files import them.
* New CI ratchet test (`apps/web/tests/unit/admin-shell-deprecation.test.ts`) that fails if the importer count of the deprecated wrappers increases. Decreases monotonically as follow-up issues land.
* Type `adminIconById` (`apps/web/components/features/dashboard/dashboard-nav/config.ts`) as `Record<AdminWorkspaceId, LucideIcon>` so TS catches missing icon entries.
* Rewrite `/app/admin/page.tsx` Overview: delete the `?view=scoreboard|workspaces` tab toggle and the 4-tile `overviewCards` array (workspaces are already in the sidebar — IA duplication). Hero row (MRR/Paying/WoW) leads, then funnel + 4-week trend + outreach/usage.
* Update `AdminScoreboardSection.tsx` — replace inline `HeroMetric` with `AnalyticsCard variant="hero"`. Section becomes funnel + trend only; caller renders hero via `AdminPage`'s hero slot.

### Out of scope (deferred to PR-2)

* `/app/admin/settings` consolidation (Platform & Env, Waitlist, Campaigns, Investors)
* Redirects from `/app/settings/admin` and `/app/admin/investors/settings`
* Migrations of Ops, Costs, People, Growth
* Route constant `ADMIN_SETTINGS` and `ADMIN_NAV_REGISTRY` addition

PR-2 blocked-by this issue.

## Why it matters

* **Inconsistent shell** across 29 admin routes — two wrappers (`AdminToolPage`, `AdminWorkspacePage`) with subtly different chrome; `AdminToolPage` wraps the page title in its own `ContentSurfaceCard` (redundant chrome).
* **Buried MRR** — the canonical "default alive" signals live behind a tab toggle on the operator's first surface. There's no single admin page where MRR is the lead.
* **Migration sprawl prevention** — the back-compat shim + CI ratchet keeps the wrappers alive only as long as it takes the autopilot loop to migrate the remaining 23 routes, after which the ratchet enforces final deletion.
