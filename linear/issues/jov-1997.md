Replace (or augment) the current tile grid in `/app/admin/ops` with an Attention Queue sorted by severity × business impact.

## Design

* List of OpsAction items, sorted: critical first, then by impact (blocks_revenue > degrades_ux > infra_risk > info)
* Each row: severity badge, source icon, title, age, status chip, owner
* Click to open detail drawer (separate issue)
* Empty state: "All clear — no active ops events"
* Loading state: skeleton rows

## Rules

* No emoji — use Lucide icons for severity and source
* No hover motion (translate/scale) — background-color change only
* Follows surface elevation rules: queue rows on `bg-surface-1`, parent page uses app-shell canvas
* No duplicate page title — `DashboardHeader` breadcrumb already shows "Ops"

## Acceptance criteria

* Component: `apps/web/components/features/admin/AttentionQueue.tsx`
* Mounted in `/app/admin/ops/page.tsx`
* Fetches from `/api/hud/actions` via TanStack Query (with staleTime + gcTime + AbortSignal)
* Typecheck + Biome pass
