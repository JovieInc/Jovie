## Follow-up

The [JOV-2269](https://linear.app/jovie/issue/JOV-2269/perfglobal-shared-css-bundle-is-103kb-brotli-likely-unused-tailwind) investigation reduced shared CSS from 77.4KB to 71.2KB brotli (7%) by excluding shell/dashboard dirs from Tailwind scan. The remaining gap to the 60KB target comes from 83 unique `bg-[color-mix(...)]` arbitrary Tailwind classes in shared components.

Each arbitrary `color-mix()` value generates 2-3x CSS output because Tailwind v4 wraps it in `@supports (color: color-mix())` blocks for browser compatibility. These classes appear in buttons, cards, toolbars, and other shared organisms that render on every route — they cannot be excluded from the Tailwind scan.

**Current state:** 71.2KB brotli
**Target:** ≤60KB brotli
**Gap:** \~11KB

**Root cause:** `tailwind.config.js` `backgroundColor` extension contains:

```js
'app-control': 'color-mix(in oklab, var(--linear-app-content-surface) 97%, var(--linear-bg-surface-0))',
'toolbar-pill': 'color-mix(in oklab, var(--linear-app-content-surface) 98%, var(--linear-bg-surface-0))',
```

These generate `bg-app-control` and `bg-toolbar-pill` utilities, plus 81 additional `bg-[color-mix()]` inline arbitrary values across 53+ components.

**Fix:** Move all `color-mix()` computed values to CSS custom properties in `design-system.css` or `globals.css`, then reference them as `var(--...)` in Tailwind config. This eliminates the `@supports` generation because CSS custom properties don't trigger the color-mix compatibility shim.

**Files to audit:** Run `grep -r 'bg-\[color-mix' apps/web --include='*.tsx' -l` — 53 files at time of investigation.

## Why it matters

11KB brotli on the shared CSS chunk = 11KB on every public pageload (homepage, /\[username\] profile, marketing pages). The profile page has no CDN warmup for cold visitors — every cold load pays this.

## Classification

Candidate

## Acceptance criteria or triage question

Should we migrate all `bg-[color-mix()]` arbitrary values to CSS custom properties to reduce shared CSS to ≤60KB brotli? Cost: touching 53+ component files across the codebase. Benefit: \~11KB savings on every public pageload.
