# JOV-2100: design(admin/growth): audit and consolidate Growth screen — remove redundant metrics

## Problem

The `/admin/growth` screen has accumulated metrics and charts that overlap with Overview and, in some cases, with the People screen. The screen doesn't have a clear, focused purpose that distinguishes it from the others.

## Goal

Audit the Growth screen and:

1. Remove metrics already visible on Overview (no duplication)
2. Remove user-count metrics that belong on People
3. Keep and improve the acquisition funnel, referral tracking, and conversion metrics — these are Growth's unique domain
4. Ensure all remaining metrics have a clear "so what" action an admin can take from this screen

## Acceptance criteria

* Growth screen shows only acquisition, referral, and conversion metrics
* No metric duplicated from Overview or People
* Charts and tables use the shared admin design system
* Screen has a clear header describing its purpose
