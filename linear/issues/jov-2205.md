# JOV-2205: [P1][Demo] Build external opportunity detector

- **Status:** Todo
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2205/p1demo-build-external-opportunity-detector

## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo shows Jovie detecting that Cosmic Gate has festival attention at EDC Las Vegas and surfacing the opportunity before Tim asks.
* Expected behavior: Jovie should ingest trusted external event and commerce signals, match them to artists, and create actionable opportunities without manual prompting.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

This is the first wow moment in the demo. Without it, the product remains reactive chat instead of an operator that finds timing-sensitive opportunities.

## Likely root cause

No production service currently owns external signal ingestion, freshness, confidence, dedupe, and artist-scoped opportunity creation.

## Acceptance criteria

* Trusted source connectors exist for event/festival and commerce-window signals.\\n- Signals include source URL, timestamp, confidence, and expiry.\\n- Duplicate signals collapse into one opportunity per artist/collaborator/time window.\\n- Opportunities can be ranked and surfaced in the app shell/chat.

## Test plan

* Unit tests for source normalization and dedupe.\\n- Integration test for EDC/Cosmic Gate-style signal to opportunity.\\n- Manual route check that an opportunity appears without user typing.\\n- Console/network checks stay clean.

## Suggested fix

Create a small opportunity-ingestion domain with source adapters, normalized signal records, dedupe keys, and an artist-scoped opportunity query consumed by chat/app surfaces.
