## Problem

Clerk proxy errors and JSON browser errors are breaking auth in production. Users can get logged out unexpectedly. The proxy.ts middleware may not be correctly decoding the FAPI host from the active publishable key.

## Acceptance Criteria

- [ ] `/__clerk` proxy correctly decodes FAPI host from the resolved publishable key at runtime
- [ ] Staging uses staging key pair and never falls back to production keys
- [ ] No JSON/Clerk errors in browser console on auth pages
- [ ] Users do not get unexpectedly logged out
- [ ] CSP allows the decoded FAPI host in connect-src, script-src, frame-src
