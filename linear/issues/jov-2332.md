# JOV-2332: P0: Configure production signup canary mailbox credentials

- **Status:** Backlog
- **Priority:** 🔴 Urgent
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2332/p0-configure-production-signup-canary-mailbox-credentials
- **Updated:** 2026-05-31T04:58:22.335Z

Production signup was restored, and PR #8907 adds a blocking production signup/onboarding canary. The remaining deployment blocker for the full real-account canary is that Doppler `jovie-web/prd` does not yet contain the dedicated mailbox credentials used by the canary.

Current redacted check:

* `E2E_PROD_SIGNUP_EMAIL_BASE`: MISSING
* `E2E_PROD_SIGNUP_PASSWORD`: MISSING
* `E2E_PROD_MAILBOX_PROVIDER`: MISSING
* `E2E_PROD_MAILBOX_CLIENT_ID`: MISSING
* `E2E_PROD_MAILBOX_CLIENT_SECRET`: MISSING
* `E2E_PROD_MAILBOX_REFRESH_TOKEN`: MISSING
* `CLERK_SECRET_KEY`: SET

Acceptance criteria:

* Add a dedicated Gmail mailbox/base address and OAuth credentials to Doppler `jovie-web/prd` for the `E2E_PROD_*` keys above.
* Trigger the production synthetic monitoring workflow manually and confirm the signup/onboarding/app-shell canary creates a real synthetic Clerk user, reads the OTP, reaches `/app`, and deletes only that plus-addressed synthetic user.
* Confirm the workflow fails if signup does not complete; warning-only signup success is not accepted.

Related PR: [https://github.com/JovieInc/Jovie/pull/8907](<https://github.com/JovieInc/Jovie/pull/8907>)
