# JOV-2340: [Infra/Cost] Eager Neon branch cleanup (delete at end of CI run, not just on PR close)

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2340/infracost-eager-neon-branch-cleanup-delete-at-end-of-ci-run-not-just
- **Updated:** 2026-05-31T04:58:18.570Z

## Why

`.github/workflows/neon-ephemeral-branch-cleanup.yml` only fires on `pull_request: closed`. PRs that stay open for days keep their ephemeral branch alive — accruing storage cost (small) and occupying branch slots (cap-relevant).

Saw this in practice: 6 ephemeral `main-{run_id}-1` branches still alive in Jovie-Primary at 2026-05-17 morning despite their CI runs being long since complete.

## What to do

Add a final `always()`-condition cleanup job to `.github/workflows/ci.yml` that deletes the Neon branch created earlier in the run, regardless of pass/fail. Keep the PR-close cleanup workflow as a safety net.

Implementation: use the same `curl DELETE` pattern already in `ci.yml`:

```yaml
- name: Delete ephemeral Neon branch
  if: always() && needs.neon-db.outputs.branch_id != ''
  run: |
    curl -sS -X DELETE \
      -H "Authorization: Bearer ${{ secrets.NEON_API_KEY }}" \
      "https://console.neon.tech/api/v2/projects/${{ vars.NEON_PROJECT_ID }}/branches/${{ needs.neon-db.outputs.branch_id }}"
```

## Acceptance

- [ ] After CI completes (pass or fail), the ephemeral branch is deleted within 1 minute
- [ ] Branch count in `Jovie-Primary` stays at a small steady-state regardless of open PR count
- [ ] PR-close cleanup workflow remains in place as fallback

## Dependency

Likely depends on or supersedes some of the work in \[Infra/Cost\] CI: skip Neon ephemeral branch for PRs that don't touch DB-affecting code.
