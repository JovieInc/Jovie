# JOV-2531: Protect investor response links from email security scanners

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2531/protect-investor-response-links-from-email-security-scanners
- **Updated:** 2026-05-31T04:57:47.371Z

Investor email action links currently use GET requests at `/investor-portal/respond?t=TOKEN&action=interested|pass` and mutate investor stage from the page handler.

Risk: mailbox/security scanners can prefetch or crawl links before a human investor clicks them, which may incorrectly mark a link as `meeting_booked` or `passed`.

Suggested direction:

* Replace direct GET mutation with a confirmation step or POST-backed action.
* Preserve token-only access, no Clerk dependency.
* Keep noindex/no-store behavior on the response route.
* Add regression coverage for scanner-like GETs that must not mutate investor state.
