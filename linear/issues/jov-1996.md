OpsAction is a separate type from AgentRunArtifact — it represents an actionable ops event derived from the `ops_events` table.

## TypeScript interface

```ts
type OpsActionSeverity = 'critical' | 'high' | 'medium' | 'low';
type OpsActionImpact = 'blocks_revenue' | 'degrades_ux' | 'infra_risk' | 'info';
type OpsActionStatus = 'queued' | 'investigating' | 'fixing' | 'verifying' | 'ready-for-pr' | 'blocked' | 'shipped' | 'dismissed';

interface OpsAction {
  id: string;
  title: string;
  summary?: string;
  severity: OpsActionSeverity;
  impact: OpsActionImpact;
  source: 'sentry' | 'ci' | 'linear' | 'deploy' | 'manual';
  status: OpsActionStatus;
  owner?: string;
  recommendedAction?: string;
  requiresApproval: boolean;
  linkedPr?: number;
  linkedIssueId?: string;
  linkedRunId?: string;
  createdAt: string;
  updatedAt: string;
}
```

## Endpoints

* `GET /api/hud/actions` — returns `OpsAction[]`, sorted by severity DESC then impact DESC, auth-gated (`isAdmin` entitlement check matching `/api/admin/re-enrich` pattern)
* `POST /api/hud/actions/:id/transition` — updates status, enforces approval gates via entitlement system

## Acceptance criteria

* Zod schema + TypeScript types in `apps/web/lib/agent-os/ops-action.ts`
* API route at `apps/web/app/api/hud/actions/route.ts`
* Auth: `getCurrentUserEntitlements()` → check `isAdmin`
* Node.js runtime (not Edge)
* Typecheck passes
