# JOV-2357: Public /pitch route reusing the investor deck (NOINDEX)

- **Status:** Todo
- **Priority:** 🟡 Medium
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** area:ui, area:marketing, enhancement, type:feature, qa:e2e
- **URL:** https://linear.app/jovie/issue/JOV-2357/public-pitch-route-reusing-the-investor-deck-noindex
- **Updated:** 2026-05-30T23:13:37.447Z

## Summary

Ship a public, NOINDEX'd `/pitch` route that reuses the existing `DeckViewer` and the 10 markdown slides at `content/investors/deck/slides/`. Make the existing investor portal and the new `/pitch` route download a static PDF (`/Jovie-Pitch-Deck.pdf`) instead of triggering `globalThis.print()`.

Slide copy stays exactly as written. Existing token-gated `/investor-portal` flow is unchanged (apart from picking up the same PDF URL).

## Scope

* Create `apps/web/app/(marketing)/pitch/page.tsx` — static (`revalidate = false`), `NOINDEX_ROBOTS`, loads slides via existing `getInvestorManifest()` + `getMarkdownDocument()`, renders `<DeckViewer slides={...} pdfUrl="/Jovie-Pitch-Deck.pdf" />`. No marketing intro section (no triple-titling).
* Modify `apps/web/app/investor-portal/_components/DeckViewer.tsx` — add optional `pdfUrl?: string` prop. When present: download button renders an `<a href={pdfUrl} download target="_blank" rel="noopener" aria-label="Download deck as PDF">`. When absent: existing `globalThis.print()` behavior preserved.
* Modify `apps/web/app/investor-portal/page.tsx` — pass `pdfUrl="/Jovie-Pitch-Deck.pdf"`.
* Add `PITCH: '/pitch'` to `apps/web/constants/routes.ts`.
* Add "Pitch" link under Company column in marketing footer.
* Generate `apps/web/public/Jovie-Pitch-Deck.pdf` once via inline Playwright/Chromium and commit it. Do NOT commit the temp script.
* Add `apps/web/tests/e2e/pitch.spec.ts` E2E: page renders, ≥1 slide title visible, PDF link present and HEAD 200, no console errors.

## Out of scope (Linear follow-ups)

* SlideShell rich slot composition (eyebrow/title/body/visual/numbers)
* Playwright PDF regeneration script + CI hook
* `/dev/deck` design preview surface
* Interactive product demo slide (smart-link release loop animation)
* Decision on `/pitch` sharing model (NOINDEX vs signed share-link)
* OG image for `/pitch`
* Per-slide rich layouts (cover/team/funding)
* Slide copy edits (intentionally locked)

## Verification

* `pnpm --filter @jovie/web run typecheck -- --pretty false`
* `pnpm biome check --write apps/web`
* `pnpm --filter web exec playwright test pitch.spec.ts`
* Visit `/pitch` and `/investor-portal` in browser, confirm deck renders, download button hits the static PDF.

## Cost Impact

* Zero recurring cost. One static page, one \~2-3MB static asset, no new external API calls, no cron, no new dependencies.
