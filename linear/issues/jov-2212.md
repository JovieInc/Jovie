# JOV-2212: [P1][Demo] Build campaign monitoring and next-move recommendations

- **Status:** Todo
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2212/p1demo-build-campaign-monitoring-and-next-move-recommendations
- **Updated:** 2026-05-31T04:57:02.381Z

## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo ends with Jovie monitoring clicks, purchases, replies, and fan activity, then recommending the next move.
* Expected behavior: Jovie should watch campaign performance after launch and produce next actions based on conversion, engagement, and external-signal persistence.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

This closes the operating loop and separates Jovie from one-shot campaign generation.

## Likely root cause

Performance data exists in separate surfaces, but campaign monitoring and next-action recommendations are not yet a production feedback loop.

## Acceptance criteria

* Campaign monitoring tracks clicks, purchases, replies, opt-ins, and channel status.\\n- Thresholds trigger typed next-move recommendations.\\n- Recommendations explain evidence and expected impact.\\n- Monitoring can be paused/resumed and audited.

## Test plan

* Unit tests for threshold logic.\\n- Integration test from mocked campaign events to next-move recommendation.\\n- Manual campaign status route check.\\n- Regression check for duplicate or repeated recommendations.

## Suggested fix

Create a campaign monitor that aggregates event streams into campaign health snapshots and emits typed next-move recommendations into chat/status surfaces.
