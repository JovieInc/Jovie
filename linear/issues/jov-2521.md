# JOV-2521: infra: Neon free-tier concurrent endpoint limit blocks Lighthouse CI on all PRs

- **Status:** Backlog
- **Priority:** 🟠 High
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2521/infra-neon-free-tier-concurrent-endpoint-limit-blocks-lighthouse-ci-on
- **Updated:** 2026-05-31T04:57:53.647Z

## Source

* Source: CI investigation during cos/design-review-perf orchestration run (2026-05-21)
* Root cause identified by analysing Lighthouse CI failures on PRs #9430, #9205, #9443

## Follow-up

The Lighthouse CI lanes (public routes, dashboard) fail at the `seed` step with HTTP 402 from Neon:

```
"You have exceeded the limit of concurrently active endpoints"
```

When 4+ PRs are in CI simultaneously, each creates an ephemeral Neon branch (Neon free tier: max 3 concurrent active endpoints). The 4th+ branch creation returns 402. The seed step fails, the Lighthouse run never completes, and all PRs show Lighthouse FAILURE — even though no actual performance budget was violated.

**This is a false negative.** The Lighthouse budget checks have never run successfully in the current CI configuration when more than 3 PRs are open simultaneously.

## Why it matters

* All 5 open PRs (#9443, #9437, #9434, #9430, #9205) currently fail Lighthouse CI
* PR merges are blocked by a non-performance infrastructure issue
* This wastes CI minutes and creates confusion about performance regressions
* Without real Lighthouse data, actual budget violations go undetected

## Options (pick one)

1. **Upgrade Neon plan** to allow more concurrent active endpoints (\~$19/mo for Launch plan, 10 concurrent endpoints) — Ship now; Re-evaluate at $200/mo MRR when $19 is meaningful
2. **Serialize Lighthouse CI** with a concurrency group so only one Lighthouse run executes at a time — free, but slower PR feedback
3. **Remove per-PR ephemeral branches** from Lighthouse and use a shared dev branch — fastest fix, but loses isolation
4. **Skip Lighthouse seed step** and use production-like static data for budget checks — cleanest for CI speed

## Classification

Required

## Acceptance criteria

* Lighthouse CI passes reliably when 4+ PRs are open simultaneously
* At least one Lighthouse run completes successfully on a PR with real CSS changes
* False-negative rate drops to 0% (only real budget violations cause Lighthouse FAILURE)

## Dependency

Affects PRs: #9443, #9437, #9434, #9430, #9205 — all currently blocked
