# JOV-1855: RELIABILITY: Every Bug Becomes a Detector

- **Project:** —
- **Labels:** codex

Turn every production failure into tests, crons, canaries, and skills. The rule for every bug fix: create or improve a detector. No bug is fixed without a new test, cron check, or canary that would have caught it.

[JOV-1848](https://linear.app/jovie/issue/JOV-1848/exhaustive-app-qa-campaign-critical-paths-and-small-pr-shipping) is the exhaustive QA campaign and execution policy. [JOV-1850](https://linear.app/jovie/issue/JOV-1850/scope-judge-should-skip-token-backed-llm-calls-when-no-api-key-is), [JOV-1849](https://linear.app/jovie/issue/JOV-1849/route-qa-should-expand-profile-alert-pay-and-releases-routes), [JOV-1837](https://linear.app/jovie/issue/JOV-1837/ci-structurally-fix-main-branch-unit-test-runtime-move-coverage-off), [JOV-1831](https://linear.app/jovie/issue/JOV-1831/new-design-smoke-lane-and-screenshot-parity-cleanup), [JOV-1835](https://linear.app/jovie/issue/JOV-1835/public-routes-lighthouse-homepage-tbt-1365ms-perf-052-username-cls-021) are existing issues attached to this loop.

**Product output:** Fewer repeat bugs; production failures caught before users; CI stays reliable.
**Velocity output:** Nightly risk scoring, flaky quarantine ledger, golden-path crons, bug-to-test rule in PR template.
**Owner profile:** coder / no_agent (crons)
**Cost route:** deterministic (crons); economy (nightly report)
**GStack skill(s):** /qa, /qa --exhaustive, /review, /ship
**Verification:** auth/onboarding/public-profile crons pass daily; flaky tests have owner+expiration; PR template requires detector for every bug
**HUD surface:** /admin/ops panel (CI health, flaky count, nightly report)
**Human approval required:** YES (CI/deploy config changes, Clerk proxy, auth changes)
