## Follow-up

The signature builder (`apps/web/lib/email-signature/build-signature.ts`) renders a "Latest release" row when `EmailSignatureInput.latestRelease = { title, url, artworkUrl }` is provided — it's covered by tests and gates `https://` only. But no current caller sources release data:

* `buildSignatureInputFromProfile` in `apps/web/lib/email-signature/profile-input.ts` doesn't accept a release argument yet.
* The admin profile shape (`AdminCreatorProfileRow`) and the dashboard `PreviewPanelData` shape (in `apps/web/app/app/(shell)/dashboard/PreviewPanelContext.tsx`) don't currently expose latest-release metadata.

To enable: extend the input adapter, pick the right release source (Spotify enrichment / DSP / manual highlight), and pass `{ title, url, artworkUrl }` from both admin + dashboard call sites.

## Why it matters

The release row is the visual hook that ties the signature to ongoing artist activity — clickable artwork + "Stream now →" is the difference between a signature and a poster. Without it the v1 signature still works but feels less alive.

## Classification

Candidate

## Acceptance criteria or triage question

Triage question: which release feed do we trust as "latest release" for a creator (Spotify enrichment top-track? a manual "featured release" set in dashboard? newest DSP match)? Pick the source, then thread it through `buildSignatureInputFromProfile`.
