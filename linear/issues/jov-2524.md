## Follow-up

The "Generate album art" suggestion pill is rendered in `apps/web/components/jovie/components/SuggestedPrompts.tsx` (sourced from `DEFAULT_SUGGESTIONS` / `FIRST_SESSION_SUGGESTIONS` in `apps/web/components/jovie/types.ts`). When `albumArtCapability.availability` is `unavailable` AND the reason is `PROVIDER_UNAVAILABLE` (provider down, not gated-by-plan), the pill should be **omitted entirely**, not just visually disabled. The `Draft album-art brief` fallback pill is already correctly seeded in this case and remains.

For `PLAN_UNAVAILABLE` the current disabled-with-tooltip behavior must remain — that's a Pro upsell surface, not a broken feature.

For `FEATURE_DISABLED` (Statsig kill switch): omit entirely.

For `unknown` (capability still loading): keep current disabled state to avoid flicker.

## Why it matters

Prod chats 2cbc704b and 91340aa5 hit `PROVIDER_UNAVAILABLE` after the user invoked album art. Even when the pill is disabled, the existence of the disabled-with-tooltip pill advertises a capability we can't deliver right now, then the user types "make album art" in free text and hits the same provider failure. Removing the advertising for genuinely-broken capabilities (vs gated ones) is the cheapest reduction in dead-end chat turns.

This is item A1 from the chat audit plan reviewed via /autoplan: Path B week-1 ship.

## Classification

Required.

## Acceptance criteria

* When `albumArtCapability.availability === 'unavailable'` and `reasonCode` is `PROVIDER_UNAVAILABLE` or `FEATURE_DISABLED`:
  * "Generate album art" pill is **not rendered** in any of the 3 layouts (rail / grid / flat).
  * "Draft album-art brief" pill **is** rendered (existing fallback logic preserved).
* When `reasonCode` is `PLAN_UNAVAILABLE`: existing behavior preserved (pill rendered, disabled, tooltip).
* When `availability === 'unknown'`: existing behavior preserved (pill rendered, disabled, tooltip).
* When `availability === 'available'`: existing behavior preserved (pill rendered, enabled; brief fallback NOT shown).
* Unit tests in `apps/web/tests/unit/chat/SuggestedPrompts.test.tsx` cover each branch above.
* Typecheck + biome + targeted vitest pass.

## Dependency

None. This is the smallest atomic step; subsequent items (A2 capability canon, A3 social-link collapse, A4 conversion instrumentation) are independent.
