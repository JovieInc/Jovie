## Follow-up

Two distinct systemic failures blocking the entire Dependabot fleet:

### Failure A: pnpm overrides drift (affects lockfile-touching PRs like #9165 react patch)

Symptom in `ESLint Server Boundaries`, `Lint`, `Typecheck` jobs:

```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH
Cannot proceed with the frozen installation. The current "overrides"
configuration doesn't match the value found in the lockfile.
```

Likely cause: `package.json` `overrides` block was updated in main without lockfile regenerating, OR Dependabot regenerates lockfile but doesn't carry forward the overrides config.

Fix path: run `pnpm install --no-frozen-lockfile` locally to regenerate, commit lockfile to main. Verify Dependabot's next refresh picks it up.

### Failure B: Neon ephemeral branch creation fails in Dependabot security context (affects A11y + Mobile Overflow lanes)

Symptom in `A11y (axe)`, `Mobile Overflow Guard (320|390|430px)`: Neon branch creation step starts and never completes / fails silently (run logs cut off after env dump). `TURBO_TOKEN` is empty in env — possible secret-scope restriction for Dependabot.

Fix path: either (a) gate Neon branch lanes to skip on Dependabot author, (b) grant Dependabot read access to the needed secrets, or (c) move these lanes off PRs entirely per [JOV-2455](https://linear.app/jovie/issue/JOV-2455/ci-subtract-lighthouse-lanes-delete-admin-pr-dashboardonboarding) (Lighthouse subtraction) — likely the same axis.

## Why it matters

8 dependency PRs piling up unmergeable. The auto-merge workflow I shipped today (.github/workflows/dependabot-auto-merge.yml) will never fire because CI never goes green. Compounds over time — more deps come in weekly.
