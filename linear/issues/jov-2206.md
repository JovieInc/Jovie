# JOV-2206: [P1][Demo] Build catalog collaborator signal matching

- **Status:** Todo
- **Priority:** None
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2206/p1demo-build-catalog-collaborator-signal-matching
- **Updated:** 2026-05-31T04:57:12.543Z

## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo connects Cosmic Gate to Tim's catalog item The Deep End.
* Expected behavior: Jovie should match external people/entities to catalog collaborators, tracks, releases, alternate spellings, and provider metadata.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

The recommendation only feels intelligent if Jovie can prove why the external signal matters to the artist's actual catalog.

## Likely root cause

Catalog metadata and collaborator aliases are not yet modeled as a reusable matching layer for opportunity detection.

## Acceptance criteria

* Collaborators are extracted from catalog imports and manual releases.\\n- Matching supports provider IDs, normalized names, aliases, and confidence.\\n- A signal involving a collaborator can resolve to relevant releases/tracks.\\n- False positives are explainable and suppressible.

## Test plan

* Unit tests for alias/name normalization.\\n- Fixture test matching Cosmic Gate to The Deep End.\\n- Regression test for repeated imports not creating duplicate matched releases.\\n- Manual app check in catalog/release detail.

## Suggested fix

Add a collaborator entity resolver backed by catalog import metadata and expose a match API for opportunity detection and chat context.
