# JOV-1984: [Profile] Releases displayed without ISRC codes — metadata gap

- **Status:** Backlog
- **Priority:** 🟠 High
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** UI Polish
- **Labels:** —
- **URL:** https://linear.app/jovie/issue/JOV-1984/profile-releases-displayed-without-isrc-codes-metadata-gap
- **Updated:** 2026-05-31T02:17:16.933Z

## Problem

Releases shown on artist profiles lack ISRC (International Standard Recording Code) information, making it impossible for DSPs and partners to properly identify and match tracks.

## Expected

ISRC codes should be displayed alongside release metadata where available, or at minimum stored and exposed in the API response.

## Where to look

* Release types/interfaces — check `PublicRelease` type definition
* Backend data pipeline for release metadata ingestion
* Profile UI components that render release info

## Notes

This is a data pipeline issue as much as a display issue. Need to trace where release metadata is fetched and whether ISRC is part of the schema.
