## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo references catalog import and release workflows created from music-industry playbooks.
* Expected behavior: Imported releases should generate release workflows with tasks for metadata, smart links, pre-save/save flows, press, playlists, content, fan notifications, merch, and reporting.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

This turns domain expertise into reusable operating system behavior and makes Jovie useful beyond one campaign moment.

## Likely root cause

Release tasks exist in pieces, but there is not yet a canonical release workflow generator driven by imported catalog state and playbooks.

## Acceptance criteria

* Imported releases generate one canonical workflow.\\n- Tasks are deduped across repeated imports.\\n- Workflow templates are versioned and auditable.\\n- The artist can see task state and generated artifacts from release detail.

## Test plan

* Unit tests for playbook expansion and dedupe keys.\\n- Integration test for repeated import not creating duplicate workflows/tasks.\\n- Manual release detail check.\\n- Migration/data backfill plan before enabling on production catalog imports.

## Suggested fix

Build a release-workflow generator with versioned playbooks, import dedupe, and task/artifact linkage to release detail surfaces.
