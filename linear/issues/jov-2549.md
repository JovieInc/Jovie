# JOV-2549: iOS internal TestFlight launch foundation

- **Status:** Todo
- **Priority:** 🟠 High
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-2549/ios-internal-testflight-launch-foundation
- **Updated:** 2026-05-31T04:56:49.743Z

## Scope

Implement the reviewed iOS launch foundation plan.

Track A:

* Full-screen dark native iOS shell with Profile QR tab and full-screen Settings.
* Browser-owned Clerk auth with hardened mobile return handling.
* Deterministic simulator/App Store screenshot workflow.
* Gated iOS download/install surfaces for alpha access.
* Release/TestFlight pipeline verification.

Track B:

* Native text chat behind the same alpha gate.
* Mobile chat API endpoints as adapters over shared chat runtime contracts.
* Cache, retry, offline queue, and contract tests.

## Acceptance

* Internal TestFlight Track A can be installed and authenticated.
* Profile QR, full-screen QR, Settings/logout, and web billing/onboarding handoff work on simulator.
* iOS marketing/download remains off for public users and gated for alpha/admin access.
* Native text chat remains behind the alpha gate until mobile chat endpoints and contract tests pass.
* Screenshots are generated deterministically and attached to the PR.

## Verification

* pnpm run ios:test
* pnpm run ios:screenshots
* mobile API unit tests
* Fastlane ios_tests/screenshots/beta where credentials allow
* gstack /qa on /download, gated install surface, and QR-opened public profile
