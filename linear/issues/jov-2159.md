## Follow-up

Extract the state-based routing decisions in `apps/web/proxy.ts` (the `userState.needsWaitlist` / `userState.needsOnboarding` / `userState.isBanned` rewrite + redirect branches around lines 810-900) into a pure function in `apps/web/lib/routing/state-routing.ts`.

Shape:

```ts
function decideRouting(input: {
  pathname: string;
  userState: ProxyUserState | null;
  isInviteRedemptionPath: boolean;
  isRSCPrefetch: boolean;
  isNavigationMethod: boolean;
  pathInfo: PathCategory;
  hasOnboardingContinuationSignal: boolean;
  redirectCount: number;
  onboardingJustCompleted: boolean;
}): { kind: 'rewrite' | 'redirect' | 'pass' | 'breaker'; target?: string };
```

`proxy.ts` becomes a thin orchestrator that calls the pure function and applies the result (rewrite / redirect / next / Sentry+next).

## Why it matters

`proxy.ts` is 1426 lines. The /start redirect-loop bug ([JOV-2147](https://linear.app/jovie/issue/JOV-2147/refactor-headernav-flyout-from-css-only-visibility-to-js-driven-hidden)) was an emergent interaction between `proxy.ts`, `app/waitlist/page.tsx`, and `ChatProposeNextStepCard.tsx`. The fix relied on adding two paths to an implicit exclusion list and adding one cookie counter. Each future state-routing change has the same shape and the same risk of forgetting a path. A pure function with exhaustive (pathname × userState) unit tests would have caught the class of bug at PR-review time.

This is also a pre-req for the next time we want to add a new authenticated route that should bypass state rewrites (e.g., a "post-checkout thank-you" page, account deletion confirmation, etc.).

  * `/start` × needsWaitlist → pass
  * `/start` × needsOnboarding → pass
  * `/onboarding/checkout` × needsWaitlist → pass
  * `/onboarding/checkout` × needsOnboarding → pass
  * `/pricing` × needsWaitlist → rewrite('/waitlist')
  * `/pricing` × needsOnboarding → rewrite('/onboarding')
  * circuit breaker at count=3 → breaker
* `proxy.ts` calls `decideRouting()` and applies the result; the existing `proxy-behavioral.test.ts` tests still pass unchanged.
* Diff is minimal in `proxy.ts` (replaces the 4 branch bodies with one switch on `kind`).
