# JOV-2097: design(admin/ops): add control panel card for operational toggles

## Problem

Operational toggles and controls (waitlist on/off, maintenance mode, feature flags, etc.) are currently scattered — some in Settings, some surfaced ad-hoc. There is no single, clear "control panel" card in the Ops screen where an admin can see and flip operational state.

## Required

Add a "Control Panel" card to `/admin/ops` that:

* Groups all operational toggles in one place (waitlist, maintenance mode, invite-only, etc.)
* Uses toggle switches (not checkboxes) for binary state
* Shows current state with a clear on/off label
* Is the canonical location for these controls (they can be removed from Settings once this lands — see companion issue)

## Design

* Card with `CardHeader` ("Control Panel") + `CardContent`
* Toggle rows: icon + label + description + toggle switch
* Compact layout, not a form
* No submit button — toggles save on change
