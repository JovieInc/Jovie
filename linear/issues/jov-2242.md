## Follow-up

Original spec asked for a user-facing `AgentRunTimeline` on /settings/connectors showing recent agent runs. CEO + DX agreed this is premature: users don't need an "AI activity feed" before they trust the AI. v1 keeps `agent_runs` data for internal logging and an admin-only `/app/admin/agent-runs/[id]` debug view — but no end-user surface.

When picking up: design the end-user timeline. Show only what's useful to a DJ ("Last extraction: 3 hours ago, 2 new shows suggested"). Hide tool-call internals, token usage, model name. Make the redaction story bulletproof.

## Why it matters

Trust pattern: surface AI work only after the user has approved or rejected enough suggested actions to have a relationship with the agent. Earlier exposure either confuses ("what is this doing?") or breaks trust ("why was this called when I didn't ask?").

## Classification

Candidate

Pickup agent must first judge whether to implement, close, or split this. Trigger: post-beta cohort feedback explicitly asks "where can I see what the AI is doing?"
