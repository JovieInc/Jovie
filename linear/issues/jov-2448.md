## Follow-up

Two releases after [JOV-2446](https://linear.app/jovie/issue/JOV-2446/sso-only-auth-layer-a-ui-canary-clerk-config-audit-cron) lands and the Clerk dashboard has been flipped to SSO-only:

1. Delete `apps/web/tests/e2e/synthetic-legacy-otp.spec.ts`
2. Delete `apps/web/tests/e2e/utils/production-signup-canary.ts`
3. Delete the unit test that covers the helper (e.g. `apps/web/tests/unit/e2e/production-signup-canary.test.ts` if it exists)
4. Remove the `Run Legacy OTP Signup Canary` step from `.github/workflows/synthetic-monitoring.yml`
5. Remove the `synthetic-legacy-otp` entry from the `resultFiles` array in the same workflow
6. Remove the `**/synthetic-legacy-otp.spec.ts` line from `apps/web/playwright.synthetic.config.ts`
7. Delete the Doppler secrets that are no longer used in `jovie-web/prd`:
   * `E2E_PROD_OTP_CHECK_URL`
   * `E2E_PROD_SIGNUP_EMAIL_BASE`
   * `E2E_PROD_SIGNUP_PASSWORD`
   * `E2E_PROD_MAILBOX_PROVIDER`
   * `E2E_PROD_MAILBOX_CLIENT_ID` and any other Gmail OAuth tokens
8. Close [JOV-2332](https://linear.app/jovie/issue/JOV-2332/p0-configure-production-signup-canary-mailbox-credentials) (mailbox creds) as superseded once these secrets are gone

## Why it matters

Keeps the codebase honest about what runs in production. The legacy spec is dead code after the dashboard flip; the secrets are unused attack surface. Two-release window (extended from one per /autoplan eng review) keeps a rollback option in case Layer A regresses unexpectedly.

## Classification

Required

## Acceptance criteria

* Repo variable `E2E_PROD_LEGACY_CANARY=0` has been set on the GitHub repo for at least one full release cycle and the Layer A workflow has been green for that cycle.
* All files listed above are deleted.
* The Doppler secrets are removed and `apps/web/scripts/check-signup-readiness.ts` runs clean against `jovie-web/prd` without them.
* [JOV-2332](https://linear.app/jovie/issue/JOV-2332/p0-configure-production-signup-canary-mailbox-credentials) is closed with a link to this PR.
