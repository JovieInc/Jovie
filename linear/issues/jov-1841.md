## Wave B2 â€” Make attribution survive confirmation

Today `notification_subscriptions` has consent fields (`smsConsentAt`, `smsConsentTextHash`, `smsConsentVersion`) but no link back to the subscribe-intent or source-link that produced it. Paid acquisition reporting and audit trails need that provenance.

## Scope

Additive migration on `notification_subscriptions`:

* `subscribe_intent_id uuid references sms_subscribe_intents(id) on delete set null`
* `source_link_id uuid references audience_source_links(id) on delete set null`
* `consent_ip text` (if not already present)
* `consent_user_agent text` (if not already present)
* `confirmed_via text` (e.g., `'sms_reply'`, `'email_otp'`, `'native_handoff'`)

Wire intent confirmation flow to populate these fields atomically with subscription creation.

## Acceptance criteria

- [ ] New migration follows immutable-migration rule (additive only).
- [ ] Subscriptions created via SMS handoff carry intent_id + source_link_id when present.
- [ ] Email confirmation flow also populates provenance where applicable.
- [ ] Consent ledger atomicity check constraint updated if needed.
- [ ] Backfill: best-effort link historical rows where intent_id or source_link_id is recoverable.
