# JOV-2124: Ops control safety — audit log and confirmation before operational control panel ships

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** area:security, area:ops
- **URL:** https://linear.app/jovie/issue/JOV-2124/ops-control-safety-audit-log-and-confirmation-before-operational
- **Updated:** 2026-05-31T02:16:12.604Z

The operational control panel issue (filed separately) will move state-changing controls to Ops. Mixing passive monitoring with state-changing controls on a command center surface requires explicit safety design first.

## Confirm before the control panel ships

1. Are high-impact toggle changes (waitlist on/off) logged to an audit trail?
2. Do destructive state changes require a confirmation step?
3. Are operational toggle permissions gated by admin role (not just admin session)?

## If no audit log exists

Add one as part of the control panel PR or as a prerequisite PR.

## Acceptance criteria

* Audit log exists or a dedicated audit log issue is filed before the control panel ships.
* High-impact toggles have confirmation dialogs.
* Role-gated access confirmed.

**This issue is a prerequisite for the operational control panel issue.**
