## Source

* Source branch/session: codex/ios-launch-foundation
* User request: implement central auth routing and icon foundation plan

## Scope

Build one auth boundary for web, iOS, and Electron so auth always returns to the originating client. Build one deterministic brand asset pipeline so iOS, Electron, PWA app icons, in-app marks, and favicons stop drifting.

## Acceptance criteria

* Auth start, callback, native exchange, client allowlists, return validation, and analytics payloads live in a shared auth-routing boundary.
* Web, iOS, and Electron auth flows include explicit client origin and cannot cross-return to another client.
* Native returns use one-time exchange codes and never place long-lived tokens in browser URLs or deep links.
* Native/public informational links open externally from iOS and Electron.
* Icons are generated from the canonical vector source with separate app-icon, in-app-mark, and favicon profiles.
* Auth boundary tests and icon contract tests pass.
