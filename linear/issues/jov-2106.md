# JOV-2106: bug(admin/touring): traffic quality filter uses disclosure pattern — replace with toggle

- **Status:** Backlog
- **Priority:** 🔵 Low
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** grok
- **URL:** https://linear.app/jovie/issue/JOV-2106/bugadmintouring-traffic-quality-filter-uses-disclosure-pattern-replace
- **Updated:** 2026-05-31T02:16:17.875Z

## Summary

The Touring screen's traffic quality filtering uses a "More info" disclosure/expansion pattern. This should be a simple toggle switch consistent with how other admin boolean controls work.

## Current behavior

Traffic quality filtering is shown/hidden via a disclosure element ("More info" or similar expand/collapse pattern).

## Expected behavior

Replace with a toggle switch that directly enables/disables the traffic quality filter in-place. The disclosure pattern adds an unnecessary interaction step for what is a simple on/off control.

## Scope

* `/admin/touring` traffic quality section
* Remove expand/collapse disclosure wrapper
* Replace with `<Switch>` / toggle component from design system
* Label should remain visible at all times (not hidden behind expand)

## Priority rationale

Low-friction UX fix; no data or logic change, purely interaction pattern.

Admin app QA/design brief, item 22.
