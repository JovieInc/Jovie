# JOV-2099: design(admin/people): migrate People tables to shared admin table system

## Problem

The `/admin/people` screen renders user tables using bespoke table markup that doesn't share styles, sorting behaviour, or pagination with the shared admin table system used elsewhere in the admin.

This creates:

* Visual inconsistency (column widths, row heights, hover states differ)
* Duplicated pagination/sorting logic
* Higher maintenance burden when the shared system changes

## Required

Refactor the People screen tables to use the shared admin table components (the same system used in the Growth and Ops screens). This is a pure code/design consistency refactor — no new columns or data changes.

## Acceptance criteria

* All People tables use the shared table component
* Sorting, pagination, and row hover behaviour match other admin tables
* Column widths and typography match the shared design
* No visible regression in data displayed
