# JOV-1998: feat(agentos): OpsAction detail drawer

- **Status:** Backlog

- **Updated:** 2026-05-31T04:59:10.248Z

Detail drawer for a selected OpsAction. Opens from the Attention Queue on row click.

## Design

* Use existing `EntitySidebarShell` — do NOT create a parallel drawer primitive
* Sections: what happened, why it matters, owner, recommended next action, linked PR/issue/run
* Approval-gate chips for actions where `requiresApproval=true` (production deploy, billing change, DB migration, auth change, outbound message, spend)
* Status transition buttons: Investigate → Fix → Verify → Ship / Dismiss
* Approval chips map to existing `@/lib/entitlements/server` entitlement check

## Acceptance criteria

* Drawer uses `EntitySidebarShell` (not a custom Sheet)
* No nested Card-inside-Card — use `variant='flat'` for inner elements
* `POST /api/hud/actions/:id/transition` wired to status chip clicks
* Typecheck + Biome pass
