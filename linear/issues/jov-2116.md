## Context

Before any IA changes are implemented, a documented inventory is needed. Current confirmed distinct jobs:

* **Overview** = KPI scoreboard (tab 1) + workspace navigation hub (tab 2) — `admin/page.tsx`
* **Ops** = live system health, deploys, AgentOS board, operational controls — `HudDashboardClient`
* **Growth** = GTM lead pipeline — `GtmFunnel` + `LeadTable` + `GtmCollapsibles` (confirmed distinct)
* **People** = user/creator/waitlist records (5 tabs with UnifiedTable)

## Deliverable

A table of "Page → primary job → data shown → overlapping fields with sibling pages → recommendation."

This table becomes the decision record for any subsequent structural changes including navigation file changes.

## Acceptance criteria

* Decision document produced and reviewed by Tim.
* No navigation file (`constants/admin-navigation.ts`) modified without this document as the authority.
