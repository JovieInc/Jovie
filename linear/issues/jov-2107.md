# JOV-2107: bug(admin): duplicate search button in admin header

- **Status:** Backlog
- **Priority:** 🔵 Low
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** —
- **Labels:** grok
- **URL:** https://linear.app/jovie/issue/JOV-2107/bugadmin-duplicate-search-button-in-admin-header
- **Updated:** 2026-05-31T02:16:16.212Z

## Summary

The admin header renders a search button even on screens that already have a search input in the main content area. This creates redundant controls that point to the same action.

## Current behavior

A search icon/button appears in the admin top-level header navigation. On pages like People (which has its own search input in the table toolbar), this means two search affordances are visible at once.

## Expected behavior

* The global header search button should only appear on screens that do not have their own contextual search input
* OR the global header search should be removed entirely in favor of per-page contextual search inputs (preferred — consistent with the admin IA consolidation direction)

## Affected screens

Any admin screen that has both the header search button and a local search input visible simultaneously.

## Priority rationale

Redundant controls create confusion about which search to use. Low risk to fix.

## Source

Admin app QA/design brief, item 21.
