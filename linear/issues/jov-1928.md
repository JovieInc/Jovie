## Follow-up

`apps/web/tests/unit/dashboard/EarningsTabTippers.test.tsx` has two failing test cases when run locally in isolation:

1. `renders tippers with formatted amounts and dates` — times out at 15s (configured timeout exceeded)
2. `renders empty state when no tippers exist` — throws "Found multiple elements with the text: Share your tip link to get started." because the previous test's DOM was not cleaned up

Root cause: the test file has `beforeEach(() => vi.clearAllMocks())` but no `afterEach` with `cleanup()`. JSDOM leaks between tests, causing duplicate text matches and timing issues. Verified that both test cases fail with main's version of `EarningsTab.tsx` as well — these are pre-existing isolation bugs unrelated to our token migration.

In CI (shard 3/6), these tests currently pass because of different parallelism/ordering. This is a timing-dependent flake risk.

Fix: add `afterEach(() => { cleanup(); })` from `@testing-library/react` to ensure DOM is reset between tests. Also investigate the 15s timeout on the tippers render test — it should not take that long and may indicate a missing mock or async issue.

## Why it matters

Pre-existing flake that could intermittently fail CI on the unit test shard. Reduces confidence in the test suite and wastes CI retries.
