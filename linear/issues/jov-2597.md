# JOV-2597: Add deterministic Promptfoo chat confirmation route contracts

- **Status:** Done
- **Priority:** 🟡 Medium
- **Assignee:** Tim White
- **Team:** JOV
- **Project:** —
- **Labels:** automated, testing, enhancement, type:feature, area:core-flows
- **URL:** https://linear.app/jovie/issue/JOV-2597/add-deterministic-promptfoo-chat-confirmation-route-contracts
- **Updated:** 2026-05-31T05:16:27.347Z

Expand the no-cost Promptfoo suite with deterministic route-contract coverage for chat confirmation endpoints after the preview-tool step.

Scope:

* Cover `/api/chat/confirm-edit`, `/api/chat/confirm-link`, `/api/chat/confirm-remove-link`, and the album-art apply confirmation route at the contract boundary.
* Keep default `pnpm run evals` deterministic and CI-safe with no model, DB, Clerk, Spotify, Stripe, or external service calls.
* Use synthetic Jovie-like data only.
* Do not change production prompts, model settings, retrieval, tool schemas, route behavior, auth, billing, or persistence.

Ship now: add deterministic confirmation-route contracts because they catch high-risk post-tool action mistakes with zero model spend or external services.
Re-evaluate when: route-level mocks diverge twice from production behavior in 30 days or a confirmation-route regression reaches staging despite these contracts.
Then: promote the relevant confirmation route to a live HTTP fixture under the existing manual/live eval lane with capped cases and no PR-CI model keys.
