# JOV-2207: [P1][Demo] Build fan segment builder from Jovie Link activity

## Evidence

* Screenshot/frame: 90-second founder demo on /demovideo
* Route: /demovideo
* Observed behavior: The demo references a warm trance fan segment from link clickers, buyers, and subscribers.
* Expected behavior: Jovie should build reusable fan segments from Jovie Link activity, purchases, subscribers, and profile interactions.

Shown cinematically in PR [JovieInc/Jovie#8671](https://linear.app/jovie/review/fixdemo-stabilize-public-demo-video-embed-82094abf613c); production implementation needed.

## Impact

Campaign recommendations need credible audience targeting, not generic advice. Segments are also key to proving the value of owning the front data layer.

## Likely root cause

Fan activity exists across profile/link/payment/notification surfaces, but there is not yet a campaign-ready segment builder with stable definitions.

## Acceptance criteria

* Segment definitions can combine genre affinity, link activity, buyers, subscribers, and recency.\\n- Segment size and sample members are visible before launch.\\n- Segments are stable, refreshable, and auditable.\\n- Missing data degrades gracefully with clear explanations.

## Test plan

* Unit tests for segment predicates.\\n- Integration test from click/buyer/subscriber fixtures to segment count.\\n- Manual dashboard check for segment preview.\\n- Privacy/access-control checks for artist-scoped data.

## Suggested fix

Introduce a fan-segmentation service over audience activity tables with saved segment definitions and campaign-scoped previews.
