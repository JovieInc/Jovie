## Follow-up

v1 only detects scope revocation on the next failed API call. If a DJ revokes Jovie's access in Google Account settings between syncs, the connector card stays in "connected" state until the next poll fails. Build a daily cron that calls `https://www.googleapis.com/oauth2/v1/tokeninfo` for each connected account and flips status to `needs_reauth` if the token is invalid.

## Why it matters

Trust + clarity. A DJ who revoked access and then opens /settings/connectors expects to see "needs to reconnect" immediately, not "connected ✓".

## Classification

Candidate

Pickup agent must first judge whether to implement, close, or split this. Lightweight (one cron, one API call per account, one DB update).

## Dependency

Blocked by C-PR-2 (need a Google connector to detect revocation for).
