# JOV-1994: AgentOS: disable stale Trigger.dev production deploy check while WDK is primary

- **Status:** Backlog
- **Priority:** 🟡 Medium
- **Assignee:** Unassigned
- **Team:** JOV
- **Project:** Local Agent Runtime
- **Labels:** codex
- **URL:** https://linear.app/jovie/issue/JOV-1994/agentos-disable-stale-triggerdev-production-deploy-check-while-wdk-is
- **Updated:** 2026-05-31T04:59:15.772Z

The post-merge check run on PR #8282's merge commit `d37a209f16e7ec17df7461b4a8810ba51c59912c` created an external Trigger.dev production deployment check that failed with `Config file not found` because the Trigger project expects `trigger.config.ts` at repo root.

Context: AgentOS v1 is intentionally WDK/Vercel Workflow-first. Trigger.dev is deferred as the fallback runtime under [JOV-1945](https://linear.app/jovie/issue/JOV-1945/agentos-phase-4-triggerdev-local-worker-poc-fallback-runtime), so main should not run a production Trigger deployment check until that fallback POC intentionally adds Trigger config and SDK wiring.

Acceptance:

* Disable or reconfigure the Trigger.dev GitHub integration for this repository while WDK is the primary durable runtime.
* Keep the check non-blocking or absent from main until [JOV-1945](https://linear.app/jovie/issue/JOV-1945/agentos-phase-4-triggerdev-local-worker-poc-fallback-runtime) intentionally lands a valid Trigger config.
* Document the integration owner and expected re-enable path on [JOV-1945](https://linear.app/jovie/issue/JOV-1945/agentos-phase-4-triggerdev-local-worker-poc-fallback-runtime).

Evidence:

* PR: [https://github.com/JovieInc/Jovie/pull/8282](<https://github.com/JovieInc/Jovie/pull/8282>)
* Failed external check: [https://github.com/JovieInc/Jovie/runs/75105012992](<https://github.com/JovieInc/Jovie/runs/75105012992>)
* Trigger deployment: [https://cloud.trigger.dev/projects/v3/proj_vweahtdptswgkifabqfb/deployments/ph9nba2y](<https://cloud.trigger.dev/projects/v3/proj_vweahtdptswgkifabqfb/deployments/ph9nba2y>)
