## Follow-up

`.github/workflows/agent-pr-verify-ready.yml` runs only on draft PRs (`if: github.event.pull_request.draft`). If the gate-evidence check fails while the PR is in draft, then the PR is later promoted to ready by some other path (e.g., `auto-pr-on-agent-push.yml`, manual `gh pr ready`, etc.), the failed check status is permanently stuck on the PR's head SHA. New `synchronize` events trigger the workflow but the job's `if:` condition is false, so it's skipped — the historical fail never clears.

Fix options to consider (pickup agent's call):

1. **Make the workflow re-run cleanly post-ready**: drop the `draft` gate, but skip evidence enforcement when PR is ready. Output a neutral or passing check_run unconditionally so the historical fail gets replaced.
2. **Have the workflow clear its own check on ready transitions**: add a `pull_request_target: types: [ready_for_review]` trigger that posts a passing/neutral `Verify Draft Agent PR` check via the Checks API.
3. **Promote the check to a required status**: the existing branch protection ruleset (Main Branch Protection) does not require it. If we required it, agent PRs without evidence would actually be blocked instead of slipping through. Combined with the codex-agent fix from [JOV-2510](https://linear.app/jovie/issue/JOV-2510/candidate-follow-up-codex-agent-must-post-agent-run-artifact-gate), this would close the gate.

## Why it matters

During this drain, 24 PRs showed a red X on this check despite being mergeable. It's noise that confuses humans and tooling. Worse, it's a paper gate — the whole point of the workflow is to require gstack QA + review + ship evidence, but right now the check fails AND the PR can still merge.

## Classification

Candidate

## Acceptance criteria

* Agent PRs that go through draft → ready without recorded evidence either:
  * Clear the failed `Verify Draft Agent PR` check (option 1 or 2), OR
  * Are actually blocked from merging (option 3 + [JOV-2510](https://linear.app/jovie/issue/JOV-2510/candidate-follow-up-codex-agent-must-post-agent-run-artifact-gate)).
* No more stuck red Xs on agent PR rows.

## Dependency

Option 3 in particular only makes sense after [JOV-2510](https://linear.app/jovie/issue/JOV-2510/candidate-follow-up-codex-agent-must-post-agent-run-artifact-gate) lands.
