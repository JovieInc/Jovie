## Follow-up

In apps/web/app/\[username\]/page.tsx, ErrorBanner is imported as a client component and included in the profile route initial bundle even though it only renders on DB error paths (rare). ErrorBanner imports sonner (toast library) and lucide-react icons, adding several chunks to every profile page load.

A server-rendered fallback component for the profile error path would eliminate these chunks from the initial load while preserving the same visual error state.

## Why it matters

Estimated reduction: 2-3 initial chunks. Low-risk: the error path is rare and does not need toast interactivity on the public profile page.

## Classification

Candidate

## Acceptance criteria or triage question

Pickup agent must first judge whether to implement, close, or split this.

If implementing: create a server-renderable error state component for the profile route error path (no hooks, no sonner dependency). Verify it renders correctly for the status=error case in page.tsx. Build and confirm chunk count drops vs baseline.

## Dependency

Depends on JOV-2271 investigation landing (PR #9190) for baseline method.
