## Source

* Current issue: ad-hoc
* Source PR: not opened yet
* Source branch/session: codex/production-signup-canary

## Follow-up

Production /start is no longer missing Turnstile env config, but Cloudflare Turnstile returns client error 110200 on [https://jov.ie/start](<https://jov.ie/start>). Add [jov.ie](<http://jov.ie>) and [www.jov.ie](<http://www.jov.ie>) to the active Turnstile widget Hostname Management allowlist, or create a dedicated production widget for [jov.ie](<http://jov.ie>) and [www.jov.ie](<http://www.jov.ie>) and update Doppler prd with the resulting NEXT_PUBLIC_TURNSTILE_SITE_KEY and TURNSTILE_SECRET_KEY.

## Why it matters

Production users still cannot start the signup/onboarding chat because Turnstile rejects the production domain. Live browser smoke on 2026-05-16: /, /signup, /signin, and /api/health/build-info are 200 with no missing-config errors; /start is 200 but renders bot challenge failed (110200). Cloudflare docs classify 110200 as Domain not authorized.

## Classification

Required

## Acceptance criteria or triage question

* [https://jov.ie/start](<https://jov.ie/start>) no longer renders turnstile is not configured.
* [https://jov.ie/start](<https://jov.ie/start>) no longer renders bot challenge failed (110200).
* Production signup canary can begin onboarding chat from homepage CTA.
* If a new production widget is used, Doppler prd and Vercel production env contain the new values without printing secrets, and production is redeployed.

## Dependency

Requires Cloudflare dashboard/API access for the Jovie account.
